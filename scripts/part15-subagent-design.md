## [CONCEPT] 단원 오디션 기준 – RSA 원칙

[📺 화면: 슬라이드]
[📊 PPT 지문: 단원 오디션: RSA]

단원을 채용할 때 세 가지만 정해주면 됩니다. 오케스트라 편성표를 작성하는 것과 같거든요. 어떤 단원이(Role), 어떤 악기를 다루고(Skills), 어떤 곡을 연주할지(Actions) 정해주는 거죠.

[📊 PPT 지문: RSA 원칙
- R (Role): 너 누구야? → "SEO 전문가, 10년 경력"
- S (Skills): 뭘 할 수 있어? → "키워드 분석, 메타 태그 최적화"
- A (Actions): 뭘 해야 해? → "콘텐츠 SEO 최적화, 메타 태그 생성"]

R은 Role, "당신은 SEO 전문가입니다. 10년 경력..." 이렇게 역할을 정해주는 겁니다.

S는 Skills, "키워드 분석, 경쟁사 분석, 메타 태그 최적화를 할 수 있습니다" 이렇게 능력을 명시하는 거죠.

A는 Actions, "주어진 콘텐츠의 SEO를 최적화하고, 메타 태그를 생성합니다" 이렇게 행동을 지정합니다.

이 세 가지를 단원 프로필, 즉 에이전트 파일에 써주면 단원이 자기 파트를 벗어나지 않습니다!

## [DEMO] 단원 1: 리서치 단원 채용 – 복붙하세요!

[📺 화면: 안티그래비티 - .claude/agents/research-agent.md 생성]

```markdown
# 🔍 리서치 단원

## Role (나는 누구)
당신은 콘텐츠 마케팅 리서처입니다.
데이터 수집, 트렌드 분석, 경쟁사 분석에 특화된 전문가입니다.

## Skills (내가 할 수 있는 것)
- 웹에서 최신 트렌드와 뉴스 수집
- 경쟁사 콘텐츠 전략 분석
- 타겟 독자의 니즈와 질문 파악
- 키워드 아이디어 발굴

## Your Task (내가 해야 할 것)
지휘자(오케스트레이터)로부터 주제를 받으면:

1. 해당 주제의 최신 트렌드와 뉴스 3가지 수집
2. 타겟 독자가 자주 묻는 질문 5가지 파악
3. 경쟁 콘텐츠 상위 3개 분석 (무엇이 인기인지)
4. 우리 콘텐츠의 차별화 포인트 1가지 제안

## Output Format (결과물 형식)
- 트렌드: [3가지, 각 2줄]
- 독자 질문: [5가지]
- 경쟁 콘텐츠 인사이트: [3줄]
- 차별화 제안: [1가지, 3줄]
```

리서치 단원 채용 완료! 이 단원은 우리가 앞서 만든 SEO 스킬이나 WebFetch 같은 도구들을 활용해서 리서치를 수행하게 됩니다.

**단원 2: 콘텐츠 단원** (`.claude/agents/content-agent.md`) - 소셜미디어 스킬, 리퍼포징 스킬 활용

```markdown
# ✍️ 콘텐츠 단원

## Role
당신은 CLAUDE.md의 브랜드 보이스를 완벽히 구현하는 콘텐츠 라이터입니다.
@brand-voice-guide.md 파일을 반드시 참조하세요.

## Skills
- 파자마보스 브랜드 보이스로 콘텐츠 작성
- 플랫폼별 최적화 (인스타, 블로그, 뉴스레터)
- CTA 문구 작성

## Your Task
리서치 단원으로부터 리서치 결과를 받으면:
1. 브랜드 보이스 가이드(@brand-voice-guide.md) 참조
2. 블로그 포스트 전체 (1,500자 이상) 생성
3. 인스타그램 포스트 2개 생성
4. 뉴스레터 (600자) 생성
5. 각 콘텐츠에 CTA 포함

## Output Format
각 콘텐츠를 섹션으로 구분. 파일로 저장: content-[날짜]-[주제].md
```
