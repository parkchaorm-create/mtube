## [CONCEPT] 자기 개선 루프 구조

[📺 화면: 슬라이드]
[📊 PPT 지문: Karpathy 자기 개선 루프
1. 콘텐츠 발행 (오늘)
2. 24시간 후 조회수 수집 (Meta Graph API)
3. 이진 Eval로 채점 (10개 Yes/No)
4. 높은 점수 + 높은 조회수 = 검증된 승리 패턴
5. 승리 패턴 기반으로 프롬프트 자동 개선
→ 다음 콘텐츠에 반영 → 1번으로 돌아감]

루프가 어떻게 도는지 설명드리겠습니다.

오늘 콘텐츠를 발행합니다.
24시간이 지나면 조회수 데이터를 수집하고요.
Meta Graph API로 인스타그램, 페이스북에서
실제 조회수를 가져오는 거죠.

그다음 이진 Eval이라는 걸로 콘텐츠를 채점합니다.
이건 바로 설명드리겠습니다.

채점 결과랑 실제 조회수를 비교하거든요.
높은 점수를 받은 콘텐츠가 실제로 조회수도 높으면요?
검증된 승리 패턴입니다.

반대로 점수는 높았는데 조회수가 낮으면요?
채점 기준이 잘못된 거니까 기준을 수정합니다.

이 분석을 바탕으로 프롬프트를 자동 개선하고요.
다음 콘텐츠를 만들 때 개선된 프롬프트를 사용하는 겁니다.

이 루프가 매일 돌아갑니다.
매일 조금씩, 자동으로 좋아지는 거죠.

## [CONCEPT] 이진 Eval – 감이 아니라 데이터

[📺 화면: 슬라이드]
[📊 PPT 지문: 이진 Eval 10개 질문 (예시)
1. 훅이 결과/변화를 묘사하는가? (기능 나열 아닌)  → Yes/No
2. 훅에 사람이나 이야기가 등장하는가?              → Yes/No
3. 쇼츠가 "할 수 있는 것" 프레임인가?              → Yes/No
4. 보도자료 톤을 피했는가?                         → Yes/No
5. 첫 프레임이 스크롤을 멈추게 하는가?              → Yes/No
6. CTA가 자연스럽게 녹아 있는가?                   → Yes/No
7. 구체적 숫자/데이터가 포함돼 있는가?              → Yes/No
8. 타깃 독자의 고통점을 건드리는가?                 → Yes/No
9. 3초 안에 핵심 가치가 전달되는가?                 → Yes/No
10. 공유하고 싶은 충동이 드는가?                    → Yes/No]

이진 Eval이 뭐냐고요?
Yes 아니면 No.
딱 두 가지로만 답하는 10개 질문입니다.

왜 이렇게 하냐면요,
"이 콘텐츠 매력적인가요?"라고 물으면 주관적이잖습니까.
"7점... 아니 6점?" 이런 식으로 흔들립니다.
감으로 매기는 거죠.

근데 "훅이 결과를 묘사하는가?" Yes or No.
"첫 프레임이 스크롤을 멈추게 하는가?" Yes or No.
이렇게 물으면 명확합니다.

10개 질문에 Yes가 8개면 8점.
6개면 6점.
기계적으로 채점이 되는 거죠.

핵심은 **감을 데이터로 바꾸는 것**입니다.
감은 주관적이지만 데이터는 객관적이거든요.

## [DEMO] 조회수 데이터 수집

[📺 화면: 안티그래비티 - auto-research.md 파일]

Meta Graph API로 실제 조회수를 가져오는 방법을 알려드리겠습니다.

Meta Graph API는 무료입니다.
인스타그램이랑 페이스북의 데이터를 가져올 수 있죠.

Meta for Developers 사이트에서 앱을 만들고,
Graph API Explorer에서 인스타 기본 권한을 추가하면 됩니다.

스킬 파일을 만들겠습니다.
`.claude/commands/auto-research.md`입니다.

```markdown
# 자기 개선 루프 스킬

## 매일 수행할 작업

### 1단계: 데이터 수집
- Meta Graph API로 최근 30개 게시물의 조회수 수집
- 상위 5개 (Top Performers)와 하위 5개 (Bottom Performers) 분류

### 2단계: 이진 Eval 채점
- 각 게시물의 원본 스크립트를 10개 Yes/No 질문으로 채점
- Eval 점수와 실제 조회수를 매칭

### 3단계: 패턴 분석
- 높은 Eval + 높은 조회수 = 검증된 승리 패턴
- 높은 Eval + 낮은 조회수 = Eval 기준 수정 필요
- 낮은 Eval + 높은 조회수 = 숨겨진 성공 요인 탐색

### 4단계: 프롬프트 개선
- auto-research-prompt.json 파일 업데이트
- 변경 사항과 근거를 research-log.md에 기록

### 5단계: 리포트
- 일일 리포트 생성: 상위/하위 콘텐츠, 프롬프트 변경 내역
```

## [CONCEPT] 리서치 로그 = 가장 가치 있는 자산

[📺 화면: 슬라이드]
[📊 PPT 지문: 리서치 로그의 가치
Day 1: 훅 프레이밍을 발표형 → 호기심 유발형으로 변경
Day 3: 개인화 언어 강화, 생활 밀착 표현 추가
Day 5: 훅 범위를 이벤트형 → 보편적/시대초월적으로 확장
Day 7: "비밀이 곧 밝혀진다" 프레이밍 추가
→ 7일간 7번의 의미 있는 진화. 전부 기록으로 남음]

카르파티가 이런 말을 했습니다.
**"리서치 로그가 가장 가치 있는 자산일 수 있다."**

왜냐면요,
모든 변경사항이 데이터와 함께 기록되니까요.
이 프롬프트를 왜 바꿨는지,
바꾸고 나서 결과가 어떻게 변했는지,
전부 남아 있습니다.

수동으로 프롬프트를 조정하면
"저번에 뭘 바꿨더라?" 기억이 안 나잖습니까.
리서치 로그에는 전부 기록돼 있거든요.

그리고 더 강력한 AI 모델이 나오면요?
이 리서치 로그를 넘겨주면 됩니다.
새 모델이 이전 모델이 멈춘 바로 그 지점에서
이어서 개선하는 거죠.

사람이 퇴사해도 인수인계가 되는 것처럼,
AI 모델이 바뀌어도 인수인계가 되는 겁니다.
