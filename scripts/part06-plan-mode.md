## [HOOK] 오프닝 후크

[📺 화면: 웹캠 단독]

보스님들, 삽질 방지 보험 첫 번째입니다.

저 처음에 Claude Code 쓸 때 이런 경험 했어요.

"소셜미디어 자동화 스킬 만들어줘"라고 했더니, Claude가 막 파일을 20개씩 쭉쭉 만들어버리는 거예요.

근데 제가 원하던 게 아닌 거예요. 수정하려고 하니까 이미 다 얽혀있고, 처음부터 다시 해야 하는 상황이 됐어요.

이게 뭔가 하면요, 오케스트라가 리허설 없이 바로 본공연을 시작한 거예요! 지휘자가 "자, 시작!"했는데 바이올린은 모차르트 하고, 첼로는 베토벤 하고 있는 거죠.

리허설 한 번만 했으면 이런 일이 없었을 거예요.

Plan Mode가 바로 그 리허설입니다. 지금 딱 알려드릴게요.

## [CONCEPT] Plan Mode vs Edit Mode

[📺 화면: 슬라이드 → 웹캠으로 설명]

[📊 PPT 지문: Edit Mode(기본) = 바로 본공연 / Plan Mode = 리허설 먼저, OK하면 본공연 시작]

Claude Code에는 기본적으로 두 가지 모드가 있어요.

기본 모드는 Edit Mode예요. Claude가 바로 파일을 만들고 수정합니다. 빠르지만 원하지 않는 결과가 나올 수 있어요. 간단한 수정 작업에는 적합해요.

Plan Mode는 리허설 먼저 하는 모드예요. Claude가 먼저 "이렇게 연주할게요, 맞죠?"라고 계획을 보여줍니다. 악보 확인 후 승인하면 그때 본공연을 시작하는 거예요. 복잡한 작업이나 새로운 악기를 만들 때 필수예요.

쉽게 말하면, Plan Mode는 리허설 먼저 보여주고 괜찮으면 본공연 시작하는 모드예요.

"아니 그러면 매번 Plan Mode 써야 해요?"

아니요! 간단한 수정은 바로 본공연, Edit Mode로 해도 돼요. 새 악기 만들거나 복잡한 일 할 때만 리허설, Plan Mode를 하면 됩니다.

## [DEMO] Plan Mode 켜는 방법 – 따라해보세요

[📺 화면: PowerShell — Claude Code 실행 중]

Plan Mode 켜는 방법은 세 가지예요.

첫 번째, 가장 쉬운 방법은 그냥 말로 하는 거예요. Claude Code 입력창에 이렇게 입력해보세요.

```
소셜미디어 자동화 스킬을 만들고 싶어.
근데 바로 만들지 말고, 계획을 먼저 보여줘. 내가 OK하면 그때 실행해.
```

"계획 먼저 보여줘"라는 한마디면 됩니다. 이게 제일 쉬워요.

두 번째 방법은 슬래시 명령어예요. Claude Code 입력창에 이렇게 입력하면 됩니다.

```
/plan
```

슬래시 plan이라고 치고 엔터를 누르면 Plan Mode가 켜져요.

세 번째 방법은 VS Code 화면을 이용하는 거예요. VS Code 아래쪽을 보시면 "Plan" 또는 "Edit"이라는 글자가 보이는 토글 버튼이 있어요. 클릭 한 번으로 모드 전환이 돼요.

## [DEMO] 리허설 있을 때 vs 없을 때 – 직접 비교

[📺 화면: PowerShell — Claude Code 실행 중]

직접 비교해볼게요. 따라하시면서 차이를 느껴보세요.

먼저 리허설 없이 바로 본공연하는 경우예요. Claude Code 입력창에 이렇게 입력해보세요.

```
소셜미디어 콘텐츠 자동화 스킬 만들어줘
```

[📺 화면: Claude Code — 바로 파일 생성 시작하는 응답]

보이시죠? Claude가 바로 파일들을 만들기 시작해요. 어디로 가는지 모르겠고, 중간에 방향 바꾸기가 어려워요.

이번엔 리허설 먼저 하는 경우예요. Claude Code에서 "/clear"를 입력해서 이전 대화를 초기화하고요.

```
/clear
```

그리고 이렇게 입력해보세요.

```
소셜미디어 콘텐츠 자동화 스킬을 만들 거야.
인스타그램과 유튜브 쇼츠 콘텐츠를 생성하는 스킬이야.
계획을 먼저 보여줘. 내가 OK하면 실행해.
```

[📺 화면: Claude Code — 계획 목록 응답]

보시죠? 이렇게 계획이 먼저 나와요. 어떤 파일을 어디에 만들 것인지, 어떤 기능을 넣을 것인지 목록으로 보여주죠.

여기서 보스님들이 "2번에서 링크드인도 추가해줘"라고 할 수 있어요. Claude Code 입력창에 이렇게 쓰면 됩니다.

```
좋아, 근데 링크드인 포스트도 추가해줘.
```

[📺 화면: 수정된 계획 응답]

그러면 Claude가 계획을 수정해서 다시 보여줍니다. 이번엔 OK라고 입력해볼게요.

```
OK, 실행해줘.
```

[📺 화면: 계획대로 파일 생성 시작]

이제 그제서야 계획대로 실행이 시작돼요. 처음부터 원하는 결과물이 나오는 거예요!

리허설 한 번으로 삽질 80% 제거!

## [CONCEPT] 언제 리허설하고, 언제 바로 공연하나?

[📺 화면: 슬라이드 → 웹캠]

[📊 PPT 지문: Plan Mode 필수 상황(새 악기 만들 때, 여러 파일 영향, 복잡한 설계) vs Edit Mode OK 상황(단순 수정, 한두 줄 변경, 간단한 질문)]

규칙 딱 하나만 기억하세요.

Plan Mode 리허설이 필수인 상황이 있어요. 새로운 악기, 즉 스킬 파일을 처음 만들 때요. 여러 파일에 영향을 줄 때요. 에이전트 같은 복잡한 걸 설계할 때요. 그리고 "왜 이렇게 됐지?" 싶은 경우예요.

반대로 바로 본공연, Edit Mode로 해도 되는 상황도 있어요. 단순 문장 수정할 때요. 기존 파일 한두 줄만 바꿀 때요. 간단한 질문에 답 받을 때요.

한 줄 규칙으로 외우세요. "처음 만드는 거면 리허설!"

## [DEMO] 실수 복구 – 되돌리기 기능

[📺 화면: PowerShell — Claude Code 실행 중]

보너스예요! 리허설 없이 공연했다가 망했을 때 대처법도 있어요.

Claude Code 입력창에 이렇게 입력해보세요.

```
/rewind
```

이렇게 하면 Claude Code가 최근 변경사항을 취소해줍니다. 공연 중에 틀린 음이 나왔을 때 "다시!"하고 처음으로 돌아가는 것과 같아요.

또는 키보드에서 Esc 키를 두 번 눌러도 현재 작업을 중단할 수 있어요.

겁먹지 마세요. 실수해도 되돌릴 수 있어요.
