# Part 08: 브랜드 보이스 – 나를 AI로 클론하는 피아노

## 메타 정보
- **예상 길이**: 15분
- **화면 구성**: 웹캠 → 슬라이드(브랜드보이스 개념) → 터미널(대화형 추출) → 안티그래비티(가이드 파일 확인) → 터미널(Humanize 실습) → 웹캠
- **핵심 비유**: 브랜드 보이스 스킬 = 피아노(오케스트라의 색깔을 결정하는 악기), 내 글쓰기 스타일 = 연주 스타일
- **특별 요소**: 대화형 브랜드보이스 추출 (파일 직접 작성 X), Humanize 프롬프트, 데이터 패킷 추출

## [HOOK]

[📺 화면: 웹캠 단독]

보스님들, 솔직히 AI가 쓴 글 딱 느낌으로 아시잖아요.

너무 매끄럽고요, 너무 균형 잡혀 있고요, 왠지 내가 안 쓴 것 같은 느낌.

"이거 AI가 쓴 거 아니야?"라는 소리 들으면 콘텐츠 생명이 끝나는 거거든요.

이게 왜 그러냐면요, AI에게 내 연주 스타일을 안 가르쳐줬기 때문입니다.

피아노가 왜 오케스트라에서 중요하냐면요, 전체 색깔을 결정하거든요. 같은 곡이라도 피아니스트 스타일에 따라 완전히 다른 느낌이 나잖아요. 조성진이 치는 쇼팽과 다른 사람이 치는 쇼팽은 색깔이 다릅니다.

오늘 만들 브랜드 보이스 스킬은요, AI가 내 글쓰기 색깔을 완벽하게 파악하게 만드는 악기입니다. 근데 이번에는 파일을 직접 작성하지 않거든요. Claude한테 질문을 받으면서 대화로 만들어낼 겁니다.

## [CONCEPT] 왜 대화형으로 만드나?

[📺 화면: 슬라이드 → 웹캠]

[📊 PPT 지문: 브랜드 보이스 구축 방식 비교
| 방식 | 직접 작성 | 대화형 추출 |
|------|---------|-----------|
| 방법 | 내가 가이드 파일을 씀 | Claude가 질문, 내가 대답 |
| 장점 | 빠름 | 놓치는 부분 없이 깊게 추출 |
| 단점 | 내가 모르는 내 특징은 빠짐 | 시간이 좀 더 걸림 |
| 결과 | 표면적 가이드 | DNA 수준의 가이드 |]

보통 브랜드 보이스 가이드를 만들라고 하면 "나는 이런 톤이고, 이런 단어를 써" 하고 직접 적잖아요. 근데 문제가 있거든요. 내가 모르는 내 특징이 있습니다.

내가 글 쓸 때 무의식적으로 하는 패턴이고요, 자주 쓰는 문장 구조고요, 특유의 리듬감 같은 건 스스로 알아채기 힘들거든요.

그러니까요 Claude한테 질문을 받는 겁니다. Claude가 "보스님은 글을 시작할 때 주로 어떤 방식을 쓰세요?", "유머를 얼마나 섞으세요?", "독자를 부를 때 어떤 호칭을 쓰세요?" 이렇게 하나씩 물어보고요, 내가 대답하면 그걸 기반으로 가이드 파일을 자동으로 만들어줍니다.

직접 쓰면 표면적인 가이드밖에 안 되는데요, 대화로 추출하면 DNA 수준의 가이드가 나오거든요. 차원이 다릅니다.

## [DEMO] 브랜드 보이스 추출 스킬 만들기

[📺 화면: 안티그래비티에서 .claude/commands 폴더 → 우클릭 → "새 파일" → "brand-voice.md" 입력]

`.claude/commands/brand-voice.md` 파일을 만들겠습니다. 이 전체를 복붙하세요.

```markdown
---
name: 브랜드 보이스 추출
description: 대화를 통해 브랜드 보이스를 추출하고, 가이드라인 파일을 자동 생성합니다
---

당신은 브랜드 스토리텔링과 글쓰기 스타일 분석 전문가입니다.

## 실행 모드 (먼저 물어보세요)
"어떤 방식으로 브랜드 보이스를 추출할까요?"
1. 대화형 추출: 제가 질문하고, 보스님이 대답하면 가이드를 만듭니다
2. 샘플 분석: 기존에 쓴 글 3~10개를 분석하여 추출합니다

## 모드 1: 대화형 추출 (권장)
95% 확신이 들 때까지 한 번에 하나씩 질문하세요.

### 질문 순서
1단계 - 기본 정체성:
- "보스님의 사업/브랜드를 한 줄로 소개해주세요"
- "타겟 고객은 누구인가요? 나이, 직업, 고민 위주로"
- "경쟁자와 나를 구분 짓는 한 가지는 뭔가요?"

2단계 - 글쓰기 스타일:
- "글을 시작할 때 주로 어떤 방식을 쓰세요? (질문/통계/이야기/선언)"
- "독자를 부를 때 어떤 호칭을 쓰세요?"
- "유머를 얼마나 섞으세요? (0~10)"
- "절대 쓰고 싶지 않은 표현이나 단어가 있으세요?"

3단계 - 톤 & 에너지:
- "글의 전반적 에너지는 어떤가요? (차분한 vs 열정적)"
- "전문가처럼 보이고 싶으세요, 친구처럼 편하게 가고 싶으세요?"
- "글에서 가장 중요하게 생각하는 가치는 뭔가요?"

4단계 - 시그니처 확인:
- "자주 쓰는 표현이나 입버릇이 있으세요?"
- "글을 끝낼 때 주로 어떻게 마무리하세요?"

### 모든 답변 수집 후
수집된 답변을 종합하여 brand-voice-guide.md 파일을 생성하세요.

## 모드 2: 샘플 분석
샘플 제공 방법 안내:
- 블로그 포스트 URL 3~10개
- 글 텍스트 직접 붙여넣기
- @파일명으로 텍스트 파일 참조

분석 순서:
1. 문장 길이, 문단 패턴, 능동태/수동태 비율
2. 자주 쓰는 단어 TOP 20, 시그니처 표현
3. 톤 & 에너지 레벨 (1~10)
4. 글 시작/마무리 패턴
5. 이 작가만의 독특한 특징 3가지

## 출력: brand-voice-guide.md 파일 생성

# [브랜드명] 브랜드 보이스 가이드

## 한 줄 정의
[이 브랜드의 목소리를 한 문장으로]

## 핵심 성격 키워드 (5개)

## 시그니처 표현 (꼭 사용)
- "[표현1]"
- "[표현2]"

## 절대 사용 금지

## 문장 스타일 가이드

## 실제 예시 (좋은 vs 나쁜)
✅ 좋은 예: "[이 브랜드 스타일]"
❌ 나쁜 예: "[이 브랜드와 안 맞는 스타일]"
```

## [CONCEPT] Humanize 프롬프트 – AI 번역투 제거

[📺 화면: 슬라이드 → 웹캠]

[📊 PPT 지문: Humanize 프롬프트 = AI 번역투 제거기
| AI 번역투 | 자연스러운 표현 |
|----------|-------------|
| "~하는 것이 중요합니다" | "이거 진짜 중요합니다" |
| "다양한 방법이 있습니다" | "방법이 몇 가지 있거든요" |
| "효과적으로 활용할 수 있습니다" | "이렇게 쓰면 됩니다" |
| "고려해볼 필요가 있습니다" | "한번 생각해보세요" |]

브랜드 보이스 가이드를 만들었는데도요, AI 특유의 번역투가 남아 있을 수 있거든요. 이걸 잡아주는 게 Humanize 프롬프트입니다.

브랜드 보이스 가이드 파일 하단에 이 섹션을 추가하세요.

```markdown
## Humanize 규칙
- "~하는 것이 중요합니다" → "이거 진짜 중요합니다"
- "다양한 방법이 있습니다" → 구체적으로 "방법이 3가지 있거든요"
- "효과적으로 활용" → "이렇게 쓰면"
- "고려해볼 필요가 있습니다" → "한번 생각해보세요"
- 한 문장에 쉼표 2개 이상 금지
- "또한", "그러나", "따라서" 같은 접속사 최소화
- 실제 입으로 말할 수 있는 문장인지 테스트
```

이 규칙이 있으면요, AI가 만든 콘텐츠에서 "AI스러운" 냄새가 확 줄어들거든요. 꿀팁이니까 꼭 넣으세요.

## [CONCEPT] 데이터 패킷 – 전문성을 뭉치로 저장

[📺 화면: 슬라이드 → 웹캠]

[📊 PPT 지문: 데이터 패킷 = 전문성 뭉치
| 패킷 종류 | 담기는 내용 | 활용 |
|----------|-----------|------|
| 전문성 패킷 | 내 분야 지식, 경험, 사례 | 콘텐츠의 깊이 |
| 오퍼 패킷 | 상품/서비스 정보, 가격 | 자연스러운 CTA |
| 로직 패킷 | 내 사고방식, 프레임워크 | 일관된 논리 전개 |]

브랜드 보이스가 "어떻게 말하는지"라면요, 데이터 패킷은 "무엇을 말하는지"입니다.

데이터 패킷이 뭐냐면요, 내 전문성이고요, 내 오퍼고요, 내 논리 구조를 뭉치로 묶어서 저장하는 겁니다. 오케스트라로 치면 연주 스타일 외에 레퍼토리 목록까지 정리하는 거죠.

세 가지 패킷이 있거든요.

전문성 패킷은 내 분야 지식, 경험, 실제 사례를 담습니다. 이게 있으면 콘텐츠에 깊이가 생기거든요.

오퍼 패킷은 내 상품이나 서비스 정보를 담습니다. CTA를 넣을 때 자연스럽게 연결되고요.

로직 패킷은 내 사고방식과 프레임워크를 담습니다. 글의 논리 전개가 항상 일관되게 되는 거죠.

이 패킷들도 brand-voice-guide.md에 추가 섹션으로 넣으면 됩니다. 다 따로 만들 필요 없이 한 파일에 정리하면 되거든요. 간단하죠?
