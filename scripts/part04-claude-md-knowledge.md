### Knowledge 1: brand-identity.md – 브랜드 정체성

이게 있으면 Claude Code가 어떤 콘텐츠를 만들든 우리 브랜드 말투로 연주하거든요.

### Knowledge 2: platform-guidelines.md – 플랫폼별 가이드

## [CONCEPT] Claude Chat 기획 → Code 빌드 분리 패턴

[📺 화면: 웹캠 단독]

[📊 PPT 지문: Claude Chat = 기획실(브레인스토밍, 전략 수립) → Claude Code = 시공현장(파일 생성, 실행) / 기획실에서 설계도 완성 → 시공현장에서 시공]

여기서 한 가지 꿀팁을 더 알려드릴게요.

이건 제가 직접 써보면서 깨달은 건데요.

Claude Chat과 Claude Code를 분리해서 쓰면 훨씬 효율적입니다.

Claude Chat이 뭐냐면요.

claude.ai 웹사이트에서 대화하는 일반 채팅 모드입니다.

여기는 기획실이라고 생각하시면 되죠.

브레인스토밍하고, 전략 세우고, 아이디어 정리하는 곳입니다.

Claude Code는요?

시공현장이거든요.

기획실에서 만든 설계도를 실제로 짓는 곳이죠.

예를 들어 "다음 주 인스타 콘텐츠 전략 짜줘"는 Claude Chat에서 하세요.

이것저것 물어보고, 수정하고, 방향 잡는 겁니다.

대화가 길어져도 토큰 부담이 적거든요.

전략이 확정되면 그때 Claude Code에서 "이 전략대로 인스타 콘텐츠 5개 만들어줘"라고 실행합니다.

설계도가 이미 있으니까 Plan Mode 한 번이면 바로 원하는 결과물이 나오죠.

이 패턴을 쓰면요.

토큰도 아끼고 결과물 품질도 올라가거든요.
