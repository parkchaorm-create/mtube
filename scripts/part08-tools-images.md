## [DEMO] 이미지 넣는 방법 3가지 – 따라해보세요

[📺 화면: 터미널 - Claude Code 실행 중]

### 방법 1: 드래그앤드롭 (가장 쉬움!)

터미널이나 VS Code Claude 채팅창에 이미지 파일을 그냥 끌어다 놓으면 됩니다.

[📺 화면: 이미지 드래그앤드롭 시연]

이렇게 하면 이미지가 첨부되고, 그 다음 질문을 입력하면 돼요.

```
[이미지 드래그앤드롭]
이 경쟁사 사이트 스크린샷 분석해서 내가 개선해야 할 점 3가지 알려줘
```

### 방법 2: @ 참조 (프로젝트 안의 파일)

이미지 파일이 프로젝트 폴더 안에 있다면 골뱅이(@)로 부르면 돼요:

```
@competitor-screenshot.png 이 이미지 분석해줘
```

### 방법 3: URL로 넣기 (웹 이미지)

```
이 이미지 분석해줘: https://example.com/image.jpg
```

"아니 세 가지 중에 뭐를 써야 해요?"

간단해요! 파일이 컴퓨터에 있으면 드래그앤드롭, 프로젝트 폴더에 있으면 @, 인터넷에 있으면 URL. 끝이에요!

## [DEMO] PDF 분석 실습 – 따라해보세요

[📺 화면: 터미널 - PDF 파일 참조]

PDF 파일도 같은 방법으로 넣을 수 있어요.

실제로 해볼게요. 이걸 그대로 입력해보세요.

```
@product-catalog.pdf 
이 카탈로그의 주요 제품 5개를 뽑아서, 
각 제품별로 인스타그램 캡션 (150자 이내) + 해시태그 5개를 만들어줘.
파자마보스 브랜드 보이스로.
```

[📺 화면: Claude Code 분석 결과]

보이시죠? PDF 내용을 읽고 각 제품별로 콘텐츠를 만들어줍니다.

이거 수동으로 하면 몇 시간 걸릴 일인데, 몇 분 만에 끝이에요.

🎉 **참고 자료 하나 넣었더니 콘텐츠 5개가 뚝딱!**

## [CONCEPT] 툴(Tool) = 악기의 특수 주법

[📺 화면: 슬라이드 – "툴 = 악기의 특수 주법들"]

이제 "툴"이 뭔지 설명드릴게요.

Claude Code는 단순히 텍스트를 읽고 쓰는 것 외에 실제로 행동할 수 있어요. 이 행동 하나하나를 "툴"이라고 해요.

바이올린 하나로 여러 가지 주법을 쓸 수 있는 것처럼, Claude Code도 여러 가지 툴을 쓸 수 있습니다.

| 주법(툴) | 하는 일 | 마케팅 활용 예시 |
|--------|---------|---------|
| Read (읽기) | 파일 읽기 | 내 블로그 글 읽기 |
| Write (쓰기) | 파일 만들기 | 콘텐츠 파일 생성 |
| Edit (수정) | 파일 수정 | 기존 콘텐츠 일부 수정 |
| Bash (실행) | 터미널 명령 실행 | 패키지 설치, 스크립트 실행 |
| WebFetch (웹 읽기) | 웹페이지 읽기 | 경쟁사 사이트 분석 |
| Glob (검색) | 파일 검색 | 특정 확장자 파일 찾기 |
| Grep (텍스트 검색) | 텍스트 검색 | 특정 단어가 있는 파일 찾기 |

이 주법들은 우리가 직접 외울 필요 없어요. Claude Code가 필요할 때 알아서 적절한 주법을 선택합니다. 우리는 "무슨 곡 연주해줘"라고만 하면 돼요!

## [CONCEPT] 툴 사용 허가 – "이 주법 써도 될까요?"

[📺 화면: 슬라이드 → 터미널]

Claude Code 쓰다 보면 이런 메시지 볼 때 있어요.

```
Claude wants to run: npm install
[Allow] [Allow All] [Deny]
```

이게 뭐냐면, 악단이 "이 특수 주법을 써도 될까요?" 허가를 묻는 거예요.

**Allow**: 이번 한 번만 허용 – "이번 곡에서만 OK"
**Allow All**: 이 공연에서 계속 허용 – "오늘 공연은 자유롭게 해"
**Deny**: 거절 – "그 주법은 쓰지 마"

처음에는 하나씩 Allow 하면서 어떤 주법을 쓰는지 구경하세요.

익숙해지면 **Allow All** 해도 됩니다. 단, 파일 삭제하는 명령은 무조건 직접 확인하세요! "악보 태워도 될까요?"에 OK 하면 안 되잖아요.

## [DEMO] 웹사이트 분석 실습 – 따라해보세요

[📺 화면: 터미널 - WebFetch 사용]

마케터한테 특히 유용한 주법, WebFetch를 실습해볼게요.

웹사이트 URL 주면 그 사이트 내용을 읽고 분석해줘요. 이걸 따라 입력해보세요:

```
경쟁사 블로그를 분석해줘.
URL: https://경쟁사블로그.com

분석 항목:
1. 가장 많이 쓰는 콘텐츠 주제
2. 포스트 제목 패턴
3. 글 구조
4. 내 블로그와 비교해서 내가 놓치고 있는 것
```

[📺 화면: WebFetch 실행 → 분석 결과]

이걸로 경쟁사 분석을 10분 안에 끝낼 수 있어요.

이 주법(WebFetch)은 나중에 SEO 스킬에서도 많이 쓸 거예요!
