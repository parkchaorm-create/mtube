[📺 화면: 슬라이드 – "독주 vs 오케스트라"]

레스토랑으로도 비유해볼게요.

처음엔 사장님이 요리도 하고, 서빙도 하고, 계산도 하고, 청소도 해요. 손님 3명까진 가능해요.

AI도 마찬가지예요!

하나의 Claude Code에게 "리서치해줘, 콘텐츠 써줘, SEO 최적화해줘, 뉴스레터까지 써줘"를 한꺼번에 시키면, 지휘자 책상(컨텍스트)이 꽉 차고 품질이 떨어집니다.

**독주 (지금까지 했던 것):**
- 하나의 AI가 모든 것을 혼자 처리
- 장점: 간단하고 빠름
- 단점: 복잡한 작업에서 품질 저하

**오케스트라 (지금부터 만들 것):**
- 여러 전문 단원이 역할 분담
- 장점: 각 분야 전문화 → 퀄리티 UP!
- 핵심: 지휘자가 전체를 조율

## [CONCEPT] 언제 독주하고, 언제 합주하나?

[📺 화면: 슬라이드]

"파자마보스님, 그러면 이제 항상 오케스트라로 해야 해요?"

아니요! 상황에 따라 다릅니다.

✅ **오케스트라(합주) 필요한 경우:**
- 주간/월간 콘텐츠 한꺼번에 만들 때
- 리서치 → 작성 → 최적화 → 배포가 이어지는 작업
- 여러 파일이 동시에 생성되는 작업

❌ **독주로 충분한 경우:**
- 인스타 포스트 하나 빠르게 만들 때
- 기존 글 수정할 때
- 간단한 질문 답 받을 때

**규칙: "이번 작업이 여러 단계를 거치고 결과물이 여러 개면 → 합주!"**

## [CONCEPT] 에이전트가 작동하는 원리 – 심플하게!

[📺 화면: 슬라이드 – "지휘자 → 단원 흐름"]

기술적으로 어떻게 작동하냐고요? 걱정 마세요, 심플합니다.

1. 지휘자가 큰 작업을 받음 ("이번 주 콘텐츠 만들어줘")
2. 작업을 악장(파트)별로 나눔
3. 각 단원에게 자기 파트 배분
4. 단원들이 순서대로 또는 동시에 연주
5. 지휘자가 결과물 모아서 최종 확인

이걸 Claude Code 스킬 파일로 구현할 수 있어요.

"아니 이걸 제가 만들 수 있다고요?"

당연하죠! 다음 파트에서 바로 만들어봅시다. 스킬 파일 만드는 거랑 비슷해요. 이미 5개나 만드신 보스님들한테 이건 식은 죽 먹기입니다!
