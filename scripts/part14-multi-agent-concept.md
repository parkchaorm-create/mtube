## [CONCEPT] 에이전트 vs 스킬 – 단원과 악기의 차이

[📺 화면: 슬라이드]
[📊 PPT 지문: 에이전트 vs 스킬 비교]

여기서 꼭 짚고 넘어가야 할 게 있습니다.
에이전트와 스킬의 차이죠.

한 줄로 정리하면 이렇습니다.
에이전트는 판단이 필요한 전문 단원.
스킬은 반복 실행하는 악기.

[📊 PPT 지문: 에이전트 vs 스킬
- 스킬(악기) = 바이올린, 첼로 같은 도구. 누구든 집어 들고 연주할 수 있는 공유 플레이북.
- 에이전트(단원) = 바이올린 전문 연주자. 자기 역할과 전문성이 있고, 상황을 판단해서 스스로 결정을 내리는 팀원.]

비유로 설명드리면요.
스킬은 기타라는 악기 자체입니다.
코드 짚는 법, 스트로크 패턴이 정해져 있죠.
누가 들어도 같은 소리가 나거든요.

에이전트는 기타리스트입니다.
곡의 분위기를 읽고,
즉흥 연주도 하고,
"이 부분은 핑거피킹이 더 어울리겠다"라고
스스로 판단하는 거죠.

우리가 ACT 3에서 만든 소셜미디어 스킬, SEO 스킬.
이건 전부 악기였습니다.
이제 이 악기들을 전문적으로 다룰
단원을 배치하는 겁니다.

## [CONCEPT] 플래너 + 리뷰어 – 2인 체제가 핵심

[📺 화면: 슬라이드]
[📊 PPT 지문: 오케스트라 2인 체제 — 플래너(기획) + 리뷰어(QA)]

오케스트라를 구성할 때
가장 중요한 포인트가 뭐냐면요.
플래너와 리뷰어.
이 2인 체제입니다.

플래너는 기획 담당이죠.
회사로 치면 기획팀장입니다.

"이번 주 콘텐츠 주제는 이거야,
블로그 먼저 쓰고,
그다음 인스타로 쪼개고,
뉴스레터로 마무리해."
이렇게 전체 작업 순서를 짜는 역할이죠.

리뷰어는 QA 담당입니다.
품질 관리자죠.

"브랜드 보이스 맞아?
맞춤법 틀린 데 없어?
SEO 점수 괜찮아?"
이렇게 최종 결과물을 검수하는 역할입니다.

[📊 PPT 지문: 왜 2인 체제?
- 사람도 자기가 쓴 글의 오타를 못 찾음 → 다른 사람이 검토해야 함
- AI도 마찬가지. 만든 AI와 검수하는 AI를 분리해야 품질이 올라감]

왜 이렇게 하냐고요?
사람도 자기가 쓴 글에서
오타를 못 찾잖습니까.

AI도 마찬가지입니다.
만든 AI가 직접 검수하면
자기 실수를 놓치거든요.

그래서 만드는 팀과 검수하는 팀을
분리해야 합니다.
이 구조가 실무에서
품질을 확 끌어올리는 비결이죠.

## [CONCEPT] 독주 vs 오케스트라 – 왜 팀이 필요한가?

[📺 화면: 슬라이드]
[📊 PPT 지문: 독주 vs 오케스트라]

레스토랑으로도 비유해볼게요.

처음엔 사장님이 요리도 하고,
서빙도 하고, 계산도 하고, 청소도 하죠.
손님 3명까진 가능합니다.

근데 손님이 10명 넘으면요?
요리하다 서빙 놓치고,
계산 밀리고,
아무것도 제대로 못 합니다.

AI도 마찬가지입니다.
하나의 Claude Code한테
"리서치해줘, 콘텐츠 써줘,
SEO 최적화해줘, 뉴스레터까지 써줘"를
한꺼번에 시키면요,
지휘자 책상이 꽉 차면서
품질이 확 떨어집니다.

이걸 컨텍스트 포화라고 하거든요.

[📊 PPT 지문: 독주 vs 오케스트라 비교
- 독주: 1 AI → 모든 작업 → 컨텍스트 포화 → 품질 저하
- 오케스트라: 전문 단원 역할 분담 → 각자 집중 → 퀄리티 UP]

오케스트라는 다릅니다.
여러 전문 단원이 역할을 나눠서
각 분야에 집중하니까
퀄리티가 확 올라가는 거죠.

그리고 지휘자가 전체를 조율하는 겁니다.

## [DEMO] context 폴더 사전 준비 – 따라하세요!

[📺 화면: 안티그래비티 — 폴더 구조]

에이전트를 만들기 전에
먼저 해야 할 일이 있습니다.
context 폴더에 브랜드 정보를 넣는 거죠.

마우스를 프로젝트 루트 폴더로 가져가세요.
왼쪽 사이드바에서 프로젝트 이름 옆
새 폴더 아이콘을 클릭합니다.
"context"라고 입력하고 엔터.

이 폴더 안에 우리 브랜드 정보 파일을 넣겠습니다.

```
context/
├── brand-voice-guide.md    ← 브랜드 보이스 가이드 (ACT 3에서 만든 것)
├── style-guide.md          ← 톤앤매너, 금지어, 서식 규칙
├── product-info.md         ← 우리 제품/서비스 핵심 정보
└── target-audience.md      ← 타겟 독자 페르소나
```

[📺 화면: 안티그래비티 — Claude Code 채팅창]

이 파일들을 직접 만들 수도 있는데요,
Claude Code한테 시키는 게 훨씬 빠릅니다.
채팅창에 이렇게 입력하세요:

"context 폴더를 만들고, brand-voice-guide.md, style-guide.md, product-info.md, target-audience.md 파일을 생성해줘. 각 파일에 [우리 브랜드 이름]에 맞는 템플릿을 채워줘."

이게 왜 중요하냐면요.
에이전트가 작업할 때
이 파일들을 참조하거든요.

악보 없이 연주할 수 없는 것처럼,
브랜드 정보 없이는
에이전트가 제대로 된 콘텐츠를 만들 수 없습니다.

## [CONCEPT] 우리 오케스트라 편성표

[📺 화면: 슬라이드]
[📊 PPT 지문: 우리가 만들 오케스트라 편성표]

자, 우리 오케스트라 편성표를 한번 보시죠!

```
🎼 지휘자 (오케스트레이터)
│
├── 📋 플래너 (기획 에이전트) — 전체 작업 순서 설계
│
├── 🔍 리서치 단원 (리서치 에이전트)
│   └── 주제 조사, 경쟁사 분석, 트렌드 파악
│
├── ✍️ 작곡가 단원 (콘텐츠 에이전트)
│   └── 블로그, 인스타, 뉴스레터 작성
│
├── 📊 음향 엔지니어 (SEO 에이전트)
│   └── 키워드 최적화, 메타데이터 세팅
│
├── 📤 매니저 단원 (배포 에이전트)
│   └── 발행 준비, 스케줄링
│
└── 🔎 리뷰어 (QA 에이전트) — 최종 품질 검수
```

지휘자가 "오늘 주제는 'AI 마케팅'이야"라고 지시하면요,
플래너가 작업 순서를 짜고,
각 단원이 자기 전문 분야에서
최고 퀄리티로 작업하고,
마지막에 리뷰어가 전체를 검수합니다.

플래너가 시작을 잡고,
리뷰어가 마무리를 잡는다.
이 2인 체제가 오케스트라의
품질을 보장하는 핵심이죠.
