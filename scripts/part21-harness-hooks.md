## [CONCEPT] 하네스 4기둥

[📺 화면: 슬라이드]
[📊 PPT 지문: 하네스 엔지니어링 4기둥
1. 컨텍스트 파일 — CLAUDE.md, agents.md (AI의 행동 규칙서)
2. 자동 강제(Hooks) — 저장 시 자동 품질 검사, 린터, 커밋 훅
3. 접근 경계 — AI가 건드릴 수 있는 폴더/도구/명령어 범위 제한
4. 가비지 컬렉션 — 나쁜 패턴 자동 감지 + 정리]

하네스는 네 개의 기둥으로 되어 있습니다.

첫 번째 기둥, 컨텍스트 파일입니다.
CLAUDE.md나 agents.md 같은 파일이죠.
이건 앞에서 이미 만들었습니다.
"새로운 라이브러리를 도입하지 마", "반드시 이 패턴을 따라."
이런 규칙을 AI가 매번 읽는 겁니다.
한번 설정해두면 모든 작업에 자동으로 적용되니까요.

두 번째 기둥, 자동 강제입니다.
Hooks라고 하죠.
Part 20에서 알람 시계 비유로 배웠던 바로 그 기능입니다.

오늘은 여기다가 "퀄리티 게이트"를 올려놓겠습니다.
이게 뭐냐면요, 콘텐츠가 품질 기준을 통과해야만 발행되도록 막는 문입니다.
그게 퀄리티 게이트죠.

세 번째 기둥, 접근 경계입니다.
AI가 어떤 폴더는 읽기만 가능하고, 어떤 명령어는 아예 실행 불가능하게 물리적으로 차단하는 거죠.

"DB를 삭제하지 마"라고 프롬프트에 쓰는 건 부탁이잖습니까.
근데 아예 삭제 명령어를 실행 못하게 막아버리면?
그건 물리적 차단입니다.

네 번째 기둥, 가비지 컬렉션입니다.
컴퓨터에서 쓰레기 메모리를 자동으로 치워주는 것처럼, AI가 만든 나쁜 패턴을 주기적으로 감지하고 정리하는 시스템이죠.

AI는 기존 코드에 나쁜 패턴이 있으면 "아, 여기선 이렇게 하나 보다" 하고 따라합니다.
그래서 자동 청소가 필요한 겁니다.

핵심 한 줄로 요약하겠습니다.
**AI가 실수했을 때 프롬프트를 고치지 마세요. 하네스를 고치세요.**
그 실수가 구조적으로 반복 불가능하도록 시스템을 바꾸는 겁니다.

## [CONCEPT] 퀄리티 게이트 = 품질 관문

[📺 화면: 슬라이드]
[📊 PPT 지문: 퀄리티 게이트 예시
- m-dash(—) 자동 제거
- 플랫폼별 글자수 초과 검사
- 금지어 필터링
- 바이럴 스코어 8점 미만 → 자동 재작성]

퀄리티 게이트는 말 그대로 품질 관문입니다.
콘텐츠가 발행되기 전에 반드시 통과해야 하는 검문소 같은 거죠.

간단한 것부터 보겠습니다.
m-dash 아시죠?
영어에서 긴 대시 기호인데, 한국어 콘텐츠에 들어가면 어색합니다.
AI가 이걸 엄청 좋아하거든요.
퀄리티 게이트에서 자동으로 걸러내는 겁니다.

글자수도 마찬가지입니다.
인스타그램은 2,200자, 트위터는 280자.
플랫폼마다 제한이 다르잖습니까.
이것도 자동으로 체크하죠.

브랜드가 절대 쓰면 안 되는 단어도 있을 겁니다.
"저렴한", "공짜" 같은 금지어를 자동으로 걸러내는 거죠.

그리고 가장 강력한 것.
**바이럴 스코어링**입니다.

LLM한테 콘텐츠를 읽게 하고, 바이럴 가능성을 10점 만점으로 채점시키는 건데요.
8점 이상이면 통과.
8점 미만이면요?
자동으로 재작성합니다.
점수가 올라올 때까지 반복하는 거죠.

200개 넘는 바이럴 포스트를 데이터베이스로 만들어 놓으면요, AI가 그걸 참고해서 판단합니다.
"이 훅은 약하다, 이 CTA는 부족하다."
자동으로 다시 쓰죠.

## [DEMO] 퀄리티 게이트 Hooks 설정

[📺 화면: 안티그래비티 - settings.json 파일]

자, 실습 들어가겠습니다.
`.claude/settings.json` 파일을 열어주세요.

```json
{
  "hooks": {
    "PreBashCommand": [
      {
        "matcher": "curl.*api.blotato",
        "command": "node .claude/hooks/quality-gate.js"
      }
    ],
    "PostFileWrite": [
      {
        "matcher": "content/**/*.md",
        "command": "node .claude/hooks/check-mdash.js {file}"
      }
    ]
  }
}
```

[📺 화면: 마우스로 PreBashCommand 부분 가리킴]

`PreBashCommand`는 bash 명령어 실행 **직전**에 끼어드는 겁니다.
SNS에 발행하는 API 호출이 실행되기 전에, 품질 검사를 먼저 돌리는 거죠.

`PostFileWrite`는 파일이 저장된 **직후**에 실행됩니다.
콘텐츠가 저장되면 자동으로 m-dash 검사를 돌리는 거죠.

## [DEMO] m-dash 검사 스크립트

[📺 화면: 안티그래비티 - check-mdash.js 파일]

m-dash 검사 스크립트를 만들겠습니다.
`.claude/hooks/check-mdash.js` 파일을 만드세요.

```javascript
const fs = require('fs');
const file = process.argv[2];
const content = fs.readFileSync(file, 'utf8');

if (content.includes('—')) {
  const fixed = content.replace(/—/g, ' - ');
  fs.writeFileSync(file, fixed);
  console.log('m-dash 자동 교체 완료!');
} else {
  console.log('m-dash 없음. 통과!');
}
```

6줄입니다.
이 6줄이 보스님들 콘텐츠에서 m-dash를 영원히 추방하죠.

테스트해볼게요.

[📺 화면: 터미널]

콘텐츠 파일에 일부러 m-dash를 넣고 저장해보겠습니다.

파일 저장하자마자 Hook이 실행되고, "m-dash 자동 교체 완료!" 메시지가 뜹니다.
파일 다시 열어보면 m-dash가 깔끔하게 교체돼 있죠.

보스님은 아무것도 안 했습니다.
하네스가 알아서 처리한 겁니다.
말이 잘못된 길로 가려고 하면 마구가 자동으로 방향을 틀어주는 거죠.

## [CONCEPT] 하네스의 진화

[📺 화면: 슬라이드]
[📊 PPT 지문: 하네스 진화 사이클
에이전트 실수 → 새 규칙 추가 → 하네스 강화 → 같은 실수 불가능
"말이 한 번 넘은 곳에 울타리가 세워진다"]

하네스의 가장 강력한 특성이 뭐냐면요.
**진화**입니다.

에이전트가 실수할 때마다, 그 실수가 새로운 규칙이 됩니다.
린터 규칙이 추가되고, 테스트가 추가되고, 제약이 추가되죠.
시간이 지날수록 시스템이 더 견고해지는 겁니다.

말이 한 번 울타리를 넘은 곳에는 더 높은 울타리가 세워지고.
두 번 다시 같은 실수를 할 수 없게 됩니다.

오픈AI 엔지니어 세 명이 코드를 한 줄도 직접 쓰지 않고 소프트웨어 제품을 배포한 사례가 있습니다.
비결이 뭐였냐면, 이 세 사람이 코드를 쓴 게 아니라 하네스를 만든 겁니다.
AI가 안전하게 코드를 만들 수 있는 환경을 설계한 거죠.

인간은 조종하고, 에이전트는 실행한다.
여기서 조종이 바로 하네스입니다.
