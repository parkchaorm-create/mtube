| 영상 구간 | 문답법 빈도 |
|-----------|------------|

## 실습 3단 구조

모든 기술적 실습은 아래 **3단 구조**를 지켜주세요.

```
1단계: 뭘 할 건지 한 줄로 (비유 포함)
→ "지금 오케스트라 지휘자를 세울 겁니다"

2단계: 그대로 복붙 (코드블록)
→ 코드는 무조건 코드블록으로! 이것만 복사하면 됨

3단계: 결과 확인 + 축하
→ "이렇게 나오면 성공! 축하합니다!"
```

## 파트 구조 템플릿

모든 스크립트 파트는 아래 4개 섹션을 반드시 포함합니다.

```
## [HOOK] 오프닝 후크 (약 30~60초)
→ "이 섹션을 보면 ~를 할 수 있게 됩니다"
→ 시청자가 끝까지 볼 이유를 제시

## [CONCEPT] 개념 설명 (있을 경우)
→ [📊 PPT 지문] 또는 [📊 인포그래픽]으로 시각화
→ 비유 필수 (기술 용어를 일상 언어로)

## [DEMO] 실습 데모
→ [📺 화면] 지시 포함, 마우스 동선 상세 기술
→ 단계별 번호 매기기 (1단계, 2단계...)
→ 복붙 가능한 명령어/프롬프트 코드블록 포함

## [RECAP] 핵심 정리 (약 30초)
→ "지금까지 배운 것 딱 3가지만 기억하세요"
→ 3줄 이내 요약

## [BRIDGE] 다음 파트 연결 (약 15초)
→ 자연스럽게 다음 챕터로 연결
→ "다음에는 ~를 해볼 건데요, 진짜 대박입니다"
```

## 문장 구조 원칙

### 조코딩 스타일 (핵심 원칙)
- **짧고 핵심적인 문장**. 군더더기 제거. 바로 실습으로 진입.
- **어조 규칙은 활성 컨텍스트의 brand-voice.md를 따른다.**
- **개조식(단어로 끝나는 표현) 금지** — 스크립트 대사는 반드시 구어체 문장으로, 소리 내어 말할 때 자연스럽게 구성한다.
- 개조식 표현은 PPT/인포그래픽 구성용으로만 별도 표기한다.

### 자문자답 사용법
```
"그럼 여기서 질문 하나 드릴게요. [질문]? 
네, 맞습니다. [답변]. 
그래서 제가 오늘 [해결책]을 준비했습니다."
```

### 단계별 번호 매기기
```
"자, 이제 딱 세 단계만 따라오세요.
첫 번째, [행동].
두 번째, [행동].
세 번째, [행동].
이게 전부입니다!"
```

### 에너지 주입 포인트
- 새로운 기능 공개 직전: "이제부터가 진짜입니다"
- 결과물 보여줄 때: "이거 보이시죠?"
- 어려운 개념 단순화 후: "이렇게 쉬운 겁니다"

## 화면 구성 지시 방법

스크립트 내에서 화면 전환을 아래 형식으로 지시합니다.

### 화면 유형별 표기
```
[📺 화면: 터미널 전체화면]
[📺 화면: VS Code + 터미널 분할]
[📺 화면: 슬라이드 - "제목 텍스트"]
[📺 화면: 웹캠 단독]
[📺 화면: 브라우저 - claude.ai]
[📺 화면: 결과물 클로즈업]
[📊 인포그래픽: 제목 또는 설명]
[📊 PPT: 제목 또는 설명]
[📊 PPT 지문: 슬라이드 내용 상세 기술]
```

### 화면 구분 규칙
1. **개념 설명 → 인포그래픽/PPT 화면**: 핵심 개념, 단계별 워크플로우, 비유 설명은 `[📊 인포그래픽]` 또는 `[📊 PPT]`로 표기. 개조식 키워드는 이 화면용으로만 사용.
2. **실습 → 컴퓨터 실제 실행 화면**: `[📺 화면]`으로 표기. 마우스 동선까지 상세히 기술.

### 실습 화면 상세 지시 규칙
- **마우스 위치·클릭 대상을 구체적으로** 기술한다.
  - 예: "화면 왼쪽 사이드바 맨 아래 톱니바퀴 아이콘을 클릭하세요"
  - 예: "상단 메뉴바에서 Terminal을 클릭하고, New Terminal을 선택하시면 됩니다"
- **입력할 프롬프트는 `" "` 안에 정확히** 기재한다.
  - 예: 채팅창에 "오늘 블로그 주제는 AI 마케팅 자동화야"라고 입력하시면 됩니다.
- 어떤 사이트에 접속하고, 어디서 가입하고, 어디서 키를 복사하는지 등 **모든 단계를 왕초보가 따라할 수 있도록** 안내한다.
