# 파자마보스 스크립트 작성 가이드

## 브랜드 아이덴티티

**채널명**: 파자마보스 (Pajama Boss)
**화자 페르소나**: 코딩 1도 모르던 40대 문과 비전공자에서 AI로 성공한 멘토
**핵심 약속**: "저처럼 삽질 없이, 딱 따라오시면 됩니다"

## 시청자 호칭 & 화자 시점

| 항목 | 규칙 | 예시 |
|------|------|------|
| 시청자 호칭 | "보스님들" | "보스님들, 안녕하세요!" |

| 질문 후 답변 | 자문자답 스타일 | "아니 이게 왜 필요하냐고요? 잠깐만요, 설명해 드릴게요" |

## 필수 어휘 패턴

### 사용 권장 표현
- 복붙, 삽질, 꿀팁, 대박, 미친, 진짜로
- "딱 N단계만 따라오세요"
- "거창한 거 없습니다"
- "이게 전부예요, 끝이에요"
- "지갑 채워드리겠습니다"
- "확실하게 끝내드리겠습니다"
- "제가 다 삽질해봤으니까요"
- "이미 늦었다고요? 천만에요"

### 사용 금지 표현
- 전문 용어 남발 (CLI, API, daemon 등을 설명 없이 사용)
- "~해야 합니다" (명령형 대신 "~하시면 돼요" 권장)
- "사실은 복잡한데..." (진입장벽 높이는 표현)
- 영어 채널명/제품명 단독 사용 (반드시 한국어 설명 병기)

## 오케스트라 비유 체계 (전체 영상 관통 메타포)

이 영상의 핵심 메타포는 **"나만의 콘텐츠 마케팅 오케스트라 만들기"** 입니다.
아래 비유를 일관되게 사용하세요.

| 기술 요소 | 오케스트라 비유 |
|-----------|----------------|
| VS Code | 무대 🎭 |
| Node.js | 전기 배선 ⚡ |
| Claude Code | 음향 장비 🎤 / 핵심 악단 |
| CLAUDE.md | 총보 (전체 악보) 🎼 |
| 터미널 | 무대 뒤 통로 (백스테이지) |
| 스킬 (.claude/commands/) | 악기 🎵 |
| 에이전트 (.claude/agents/) | 오케스트라 단원 🎭 |
| 오케스트레이터 | 지휘자 🎼 |
| Hooks | 자동 공연 시스템 / 알람 시계 ⏰ |
| Plan Mode | 리허설 🎬 |
| Edit Mode | 본공연 |
| 컨텍스트 윈도우 | 지휘자의 책상 크기 📋 |
| 토큰 | 택시 미터기 🚕 |
| 모델 (Opus/Sonnet/Haiku) | 택시 등급 (리무진/일반/경차) |
| API | 주문 전화번호 📞 |
| Webhook | 초인종 🔔 |

### 기술 용어 비유 규칙
**용어를 빼지 마세요. 비유를 입히세요!**
기술 용어가 처음 등장할 때 반드시 **5초 비유**를 붙이세요.

예시: "API, 어려운 영어 맞죠? 쉽게 말하면 '주문 전화번호'예요."

| 영상 구간 | 문답법 빈도 |
|-----------|------------|
| 전반부 (0~2시간) | 파트당 최소 2회 |
| 중반부 (2~3시간) | 파트당 최소 2회 |
| 후반부 (3시간~) | **5분에 1회 (강화!)** |

후반부일수록 시청자 집중력이 급격히 떨어지므로 문답법을 의도적으로 늘리세요.
패턴: "아니 왜 이렇게 해야 하냐고요? → 아! 좋은 질문이에요. 왜냐면..."

## 실습 3단 구조

모든 기술적 실습은 아래 **3단 구조**를 지켜주세요.

```
1단계: 뭘 할 건지 한 줄로 (비유 포함)
→ "지금 오케스트라 지휘자를 세울 거예요"

2단계: 그대로 복붙 (코드블록)
→ 코드는 무조건 코드블록으로! 보스님들은 이것만 복사하면 됨

3단계: 결과 확인 + 축하
→ "이렇게 나오면 성공! 축하합니다, 지휘자를 세우셨어요! 🎉"
```

## 파트 구조 템플릿

모든 스크립트 파트는 아래 4개 섹션을 반드시 포함합니다.

```
## [HOOK] 오프닝 후크 (약 30~60초)
→ "이 섹션을 보면 ~를 할 수 있게 됩니다"
→ 시청자가 끝까지 볼 이유를 제시

## [CONCEPT] 개념 설명 (있을 경우)
→ 슬라이드 or 그림으로 시각화 지시 포함
→ 비유 필수 (기술 용어를 일상 언어로)

## [DEMO] 실습 데모
→ 화면 캡처 지시 포함
→ 단계별 번호 매기기 (1단계, 2단계...)
→ 복붙 가능한 명령어/프롬프트 코드블록 포함

## [RECAP] 핵심 정리 (약 30초)
→ "오늘 배운 것 딱 3가지만 기억하세요"
→ 3줄 이내 요약

## [BRIDGE] 다음 파트 연결 (약 15초)
→ 자연스럽게 다음 챕터로 연결
→ "다음에는 ~를 해볼 건데요, 진짜 대박이에요"
```

## 문장 구조 원칙

### 자문자답 사용법
```
"그럼 여기서 질문 하나 드릴게요. [질문]? 
네, 맞아요. [답변]. 
그래서 제가 오늘 [해결책]을 준비했습니다."
```

### 단계별 번호 매기기
```
"자, 이제 딱 세 단계만 따라오세요.
첫 번째, [행동].
두 번째, [행동].
세 번째, [행동].
이게 전부예요!"
```

### 에너지 주입 포인트 (✦ 표시)
- 새로운 기능 공개 직전: "이제부터가 진짜예요"
- 결과물 보여줄 때: "보스님들, 이거 보이세요?"
- 어려운 개념 단순화 후: "이렇게 쉬운 거예요. 끝이에요."
