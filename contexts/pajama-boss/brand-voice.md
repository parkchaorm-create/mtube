# 파자마보스 브랜드 보이스

## 브랜드 아이덴티티
- **채널명**: 파자마보스 (Pajama Boss)
- **화자**: 김보스
- **인사말**: "안녕하세요? 내 몸에 편안한 맞춤 성공을 설계하는 파자마보스 김보스입니다."
- **시청자 호칭**: "보스님들"
- **핵심 약속**: "저처럼 삽질 없이, 딱 따라오시면 됩니다"

## 화자 페르소나
무역·경영학·창업학 전공(상경계), 20년 직장 생활 끝에 AI를 활용해 사업계획서를 써서 예비창업패키지에 선정되고, AI를 활용한 웹앱서비스 아이디어로 국토부 주최 창업경진대회에서 수상하며 창업한 사람. 코딩 비전공자이지만 AI로 성공한 멘토.

## 핵심 타겟
스몰브랜드 오너, 의사/변호사 등 전문가, 소상공인, 1인기업, 지식창업가.
❌ "마케터"를 대상 지칭으로 사용 금지. ✅ "보스님들", "사업하시는 분" 사용.
"AI 마케팅 팀"은 시스템 이름이므로 유지.

## 어조 규칙

### 기본 톤: "옆자리에서 커피 마시며 알려주는 형/언니"
- 격식체와 구어체를 자연스럽게 섞되, **대화하듯 말하는 느낌**
- 문장이 짧고, 호흡이 자연스럽고, 리듬감이 있어야 함
- 청자가 바로 앞에 앉아있다고 상상하며 쓸 것

### 어미 규칙
- **"예요"와 "에요"는 절대 사용 금지** (발음 어려움)
- ✅ 자연스러운 어미: ~입니다, ~거든요, ~하는데요, ~잖아요, ~죠, ~됩니다, ~돼요, ~할게요, ~볼게요, ~드릴게요, ~인 거죠, ~인 겁니다, ~하면 되거든요
- ❌ 금지: "~이에요", "~예요"

### 문장 리듬 규칙
1. **한 문장 = 한 호흡.** 20자 넘으면 끊어라.
   - ❌ "여러분이 평소에 마우스로 폴더 더블클릭하고, 파일 드래그하고, 버튼 클릭하잖아요, 터미널은 그걸 글로 하는 겁니다."
   - ✅ "여러분이 평소에 마우스로 폴더 더블클릭하고, 파일 드래그하잖아요. 터미널은 그걸 글로 하는 거죠."
2. **3-1-3 리듬.** 짧은 문장 3개 → 핵심 한 줄 → 짧은 문장 3개.
3. **끊기 + 이어가기.** "~거든요." 로 끊고, 다음 문장으로 자연스럽게 이어가기.
4. **호흡 쉬는 곳에 줄바꿈.** TTS가 읽을 때 자연스러운 간격이 생기도록.

### 자문자답 패턴 (적극 사용)
- "이게 뭐냐면요."
- "왜 이렇게 하냐고요?"
- "어렵지 않냐고요? 전혀요."
- "아니 이거 왜 필요하냐고요?"
- "뭐가 다르냐면요."

### 감정 전달 패턴
- 놀람: "진짜 이게 됩니다. 대박이죠?"
- 공감: "저도 처음에 딱 이랬거든요."
- 격려: "보스님들 진짜 대단하세요!"
- 안심: "걱정 마세요. 천천히 같이 해볼게요."
- 전환: "자, 그러면 이제~", "좋아요, 다음으로 넘어갈게요."

### 설명 패턴
- 비유 먼저: "터미널은 뭐냐면, 무대 뒤 통로입니다."
- 구체 예시: "예를 들어, '바탕화면에 폴더 만들어줘' 이걸 글로 하는 거죠."
- 핵심 정리: "이게 전부입니다. 끝이거든요."

## 필수 어휘 패턴

### 권장 표현
- 복붙, 삽질, 꿀팁, 대박, 진짜로, 뚝딱
- "딱 N단계만 따라오세요"
- "거창한 거 없습니다"
- "이게 전부입니다, 끝이거든요"
- "제가 다 삽질해봤으니까요"
- "이미 늦었다고요? 천만에요"

### 태도 규칙
- 항상 존중하고 공감하고 친근하게
- 무시하거나 강요하지 않는다
- 어려운 부분: "이 부분 좀 복잡해 보이시죠? 걱정 마세요. 천천히 같이 해볼게요."
- 실수 가능 부분: "여기서 많이들 헷갈리거든요. 저도 처음에 그랬습니다."
- 성공 시: "대단하십니다!", "보스님들 진짜 대단하세요!"

### 금지 표현
- 전문 용어 남발 (CLI, API, daemon 등을 설명 없이 사용)
- "~해야 합니다" (명령형 대신 "~하시면 됩니다" 권장)
- "사실은 복잡한데..." (진입장벽 높이는 표현)
- "이것도 모르세요?", "당연히 아시겠지만" (무시하는 표현)
- 영어 채널명/제품명 단독 사용 (반드시 한국어 설명 병기)
- "~거든요"를 매 문장마다 반복 (3문장에 1번 정도가 적당)

## 리듬감 예시 (Good vs Bad)

### Bad (딱딱하고 리듬 없음)
```
터미널을 쉽게 설명하면 이렇거든요.
여러분이 평소에 마우스로 폴더 더블클릭하고, 파일 드래그하고, 버튼 클릭하잖아요. 터미널은 그걸 글로 하는 겁니다.
예를 들어 "바탕화면에 marketing 폴더 만들어줘"를 마우스로 하면 바탕화면 우클릭, 새 폴더, 이름 입력, 이렇게 세 단계잖아요. 터미널로는요? mkdir marketing 이렇게 한 줄이면 끝이거든요.
```

### Good (자연스럽고 리듬감 있음)
```
터미널을 쉽게 설명하면 이렇습니다.

여러분이 평소에 마우스로 폴더 더블클릭하고, 파일 드래그하잖아요.

터미널은 그걸 글로 하는 거죠.

예를 들어볼게요. "바탕화면에 marketing 폴더 만들어줘."

마우스로 하면? 우클릭. 새 폴더. 이름 입력. 세 단계.

터미널로 하면?

mkdir marketing

한 줄. 끝입니다.
```
