## 설명과 실습 분리 원칙 (최우선 규칙)

모든 파트에서 **[CONCEPT] 설명**과 **[DEMO] 실습**을 명확히 분리한다.

### [CONCEPT] 설명 구간
- 처음 나오는 개념, 어려운 용어, 작업 흐름 개요를 다룬다.
- **반드시 비유, 그래프, 도표, 인포그래픽**을 활용한다.
- `[📊 PPT 지문:]` 또는 `[📊 인포그래픽:]`으로 표기한다.
- 이 구간에서는 컴퓨터 화면을 보여주지 않는다.

### [DEMO] 실습 구간
- **초등학생도 시키는 대로만 따라하면 될 수준**으로 작성한다.
- 모든 동작을 빠짐없이 기술:
  - 어떤 창을 연다 (브라우저, 안티그래비티, 터미널 등)
  - 어떤 링크/URL로 접속한다
  - 해당 인터페이스의 **상하좌우 어디에 있는 어떤 버튼**을 누른다
  - **어떤 대화창**(Claude Code 채팅창, 안티그래비티 채팅창, 터미널 등)에 **어떤 프롬프트**를 입력한다
  - 입력할 내용은 `" "` 안에 정확히 기재한다
- `[📺 화면:]`으로 표기하며, 실제 컴퓨터 화면을 보여준다.

### 나쁜 예 vs 좋은 예
```
❌ 나쁜 예: "settings.json을 수정하세요"
✅ 좋은 예: "안티그래비티 왼쪽 하단의 톱니바퀴 아이콘을 클릭하세요. 
   Settings가 열리면 오른쪽 상단의 '{}' 아이콘을 클릭하시면 됩니다. 
   그러면 settings.json 파일이 열리거든요."

❌ 나쁜 예: "Claude Code에 프롬프트를 입력하세요"  
✅ 좋은 예: "안티그래비티 하단의 Claude Code 채팅창에 
   \"마케팅 팀을 만들어줘\"라고 입력하고 엔터를 누르세요."
```

## 톤 & 태도 규칙

- **항상 존중하고 공감하고 친근하게.**
- 무시하거나 강요하지 않는다.
- "이것도 모르세요?", "당연히 아시겠지만" 같은 표현 금지.
- 어려운 부분이 나올 때: "이 부분 좀 복잡해 보이시죠? 걱정 마세요. 천천히 같이 해볼게요."
- 실수할 수 있는 부분: "여기서 많이들 헷갈려하시거든요. 저도 처음에 그랬습니다."
- 성공했을 때: "대단하십니다!", "보스님들 진짜 대단하세요!"

## 문장 구조 원칙

### 조코딩 스타일 (핵심 원칙)
- **짧고 핵심적인 문장**. 군더더기 제거. 바로 실습으로 진입.
- **니다체와 요체를 섞어 쓰되, 격식 있는 톤 유지.**
- **"예요"와 "에요"는 절대 사용 금지** (발음이 어려움).
  - ✅ 사용 가능: ~입니다, ~거든요, ~하는데요, ~그렇죠?, ~죠, ~맞습니다, ~됩니다, ~돼요
  - ❌ 금지: "~이에요", "~예요" (예: "이거예요" → "이겁니다", "컴퓨터예요" → "컴퓨터입니다")
- **개조식(단어로 끝나는 표현) 금지** — 스크립트 대사는 반드시 구어체 문장으로, 소리 내어 말할 때 자연스럽게 구성한다.
- 개조식 표현은 PPT/인포그래픽 구성용으로만 별도 표기한다.
- **어조 세부 규칙은 활성 컨텍스트의 brand-voice.md를 따른다.**

### 자문자답 사용법
```
"그럼 여기서 질문 하나 드릴게요. [질문]? 
네, 맞습니다. [답변]. 
그래서 제가 오늘 [해결책]을 준비했습니다."
```

### 단계별 번호 매기기
```
"자, 이제 딱 세 단계만 따라오세요.
첫 번째, [행동].
두 번째, [행동].
세 번째, [행동].
이게 전부입니다!"
```

### 에너지 주입 포인트
- 새로운 기능 공개 직전: "이제부터가 진짜입니다"
- 결과물 보여줄 때: "이거 보이시죠?"
- 어려운 개념 단순화 후: "이렇게 쉬운 겁니다"

## 시간 표현 금지 규칙

스크립트 대사에서 **구체적 타임스탬프 언급 금지**.
하나의 긴 영상으로 편집하므로 타임스탬프가 맞지 않을 수 있습니다.

- ❌ "18분에서 자세히 해드릴게요", "28분에서 같이 해드립니다"


## 파트 구조 템플릿

모든 스크립트 파트는 아래 섹션을 반드시 포함합니다.

```
## [HOOK] 오프닝 후크 (약 30~60초)
→ "이 섹션을 보면 ~를 할 수 있게 됩니다"
→ 시청자가 끝까지 볼 이유를 제시

## [CONCEPT] 개념 설명 (있을 경우)
→ [📊 PPT 지문: ...] 으로 인포그래픽 시각화
→ 비유 필수 (기술 용어를 일상 언어로)

## [DEMO] 실습 데모
→ [📺 화면: ...] 지시 포함, 마우스 동선 상세 기술
→ 단계별 번호 매기기 (1단계, 2단계...)
→ 복붙 가능한 명령어/프롬프트 코드블록 포함
→ 입력할 프롬프트는 " " 안에 정확히 기재

## [RECAP] 핵심 정리 (약 30초)
→ [📊 PPT 지문: ...] 으로 3줄 요약 시각화
→ "지금까지 배운 것 딱 3가지만 기억하세요"

## [BRIDGE] 다음 파트 연결 (약 15초)
→ 자연스럽게 다음 챕터로 연결
→ "다음에는 ~를 해볼 건데요, 진짜 대박입니다"
→ 구체적 타임스탬프 사용 금지 ("다음 파트에서" 등으로 대체)
```

## 화면 구성 지시 방법

스크립트 내에서 화면 전환을 아래 형식으로 지시합니다.

### 화면 유형별 표기
```
[📺 화면: 안티그래비티 전체화면]
[📺 화면: 안티그래비티 + 터미널 분할]
[📺 화면: 안티그래비티 — 폴더 구조]
[📺 화면: 웹캠 단독]
[📺 화면: 브라우저 → claude.ai]
[📺 화면: 결과물 클로즈업]
[📊 PPT 지문: 슬라이드 내용 상세 기술]
[📊 인포그래픽: 제목 또는 설명]
```

### 화면 구분 규칙
1. **개념 설명 → 인포그래픽/PPT 화면**: 핵심 개념, 단계별 워크플로우, 비유 설명은 `[📊 PPT 지문:]`으로 표기. 개조식 키워드는 이 화면용으로만 사용.
2. **실습 → 컴퓨터 실제 실행 화면**: `[📺 화면:]`으로 표기. 마우스 동선까지 상세히 기술.

### 실습 화면 상세 지시 규칙
- **마우스 위치·클릭 대상을 구체적으로** 기술한다.
  - 예: "화면 왼쪽 사이드바 맨 아래 톱니바퀴 아이콘을 클릭하세요"
  - 예: "상단 메뉴바에서 Terminal을 클릭하고, New Terminal을 선택하시면 됩니다"
- **입력할 프롬프트는 `" "` 안에 정확히** 기재한다.
  - 예: 채팅창에 "오늘 블로그 주제는 AI 마케팅 자동화야"라고 입력하시면 됩니다.
- 어떤 사이트에 접속하고, 어디서 가입하고, 어디서 키를 복사하는지 등 **모든 단계를 왕초보가 따라할 수 있도록** 안내한다.
- **JSON, 코드 등 왕초보에게 어려운 작업은 Claude Code에게 시키는 방식을 먼저 안내**하고, 직접 하고 싶은 분을 위해 코드도 함께 제시한다.
