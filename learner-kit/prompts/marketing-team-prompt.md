# 전체 마케팅 팀 구축 기획서 (prompt.md)

> 이 파일은 **Part 16**에서 사용합니다.
> AI 마케팅 오케스트라의 전체 팀 구조와 실행 지침을 정의하는 마스터 프롬프트입니다.

---

```
나의 AI 마케팅 오케스트라를 구축해줘.
아래 기획서에 따라 전체 시스템을 설계하고 실행해줘.

## 오케스트라 구성

### 지휘자 (오케스트레이터)
- 역할: 전체 워크플로우 조율
- 실행 방식: /marketing-team 명령어로 전체 팀 가동
- 악장 순서: 리서치 → 콘텐츠 → SEO → 배포

### 1번 단원: 리서치 에이전트
- 역할: 주제 조사, 트렌드 분석, 경쟁사 분석
- 산출물: research-notes.md
- 핵심 역량:
  - 아웃라이어 스코어 분석
  - 타겟 고객 질문 수집
  - 키워드 기회 발굴

### 2번 단원: 콘텐츠 에이전트
- 역할: 블로그, 소셜미디어, 뉴스레터 작성
- 산출물: blog-post.md, social-posts.md, newsletter.md
- 핵심 역량:
  - 브랜드 보이스 적용 (@brand-voice-guide.md)
  - 채널별 최적 포맷 적용
  - 후크 + CTA 설계

### 3번 단원: SEO 에이전트
- 역할: 콘텐츠 검색 최적화
- 산출물: seo-optimized 버전 콘텐츠
- 핵심 역량:
  - 메타 태그 생성
  - 키워드 밀도 최적화
  - GEO(AI 검색) 최적화

### 4번 단원: 배포 에이전트
- 역할: 채널별 발행 준비 및 일정 관리
- 산출물: distribution-ready.md
- 핵심 역량:
  - 채널별 포맷 변환
  - 발행 체크리스트 생성
  - 최적 발행 시간대 추천

## 실행 워크플로우

### 입력
- 주제: [이번 주 콘텐츠 주제]
- 타겟 채널: [전체 / 특정 채널]
- 발행일: [예정일]
- 참고 자료: [URL, 파일 등]

### 실행 순서
1악장 (리서치) → 리서치 결과 확인 → 승인
2악장 (콘텐츠) → 초안 확인 → 피드백 반영
3악장 (SEO) → 최적화 확인 → 승인
4악장 (배포) → 최종 검토 → 발행 준비 완료

### 출력
모든 결과물은 content/[날짜]/ 폴더에 저장:
├── research-notes.md
├── blog-post.md
├── social-posts.md
├── newsletter.md
├── seo-report.md
└── distribution-ready.md

## 품질 기준
- 브랜드 보이스 일치도: 95% 이상
- SEO 점수: 7/10 이상
- CTA 포함: 모든 콘텐츠에 필수
- AI 번역투: 0%
- 팩트 체크: 모든 통계/인용에 출처 명시

이 기획서를 기반으로 전체 시스템을 구축하고,
첫 번째 공연(테스트 실행)을 진행해줘.
```

---

## 사용 팁

- 이 프롬프트는 처음 오케스트라를 구축할 때 한 번 사용합니다.
- 구축 후에는 `/marketing-team [주제]` 명령어로 간단히 실행하세요.
- 에이전트 파일은 `.claude/agents/` 폴더에 자동 생성됩니다.
- 각 에이전트의 성능은 사용하면서 스킬 파일을 수정해 개선하세요.
