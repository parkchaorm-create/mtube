## 1. 컨텍스트 초기화 (가장 중요!)

### 언제 `/clear` 해야 하나?

| 상황 | 신호 | 조치 |
|------|------|------|
| 응답이 이상해짐 | AI가 브랜드 톤 무시, 이전 지시 잊어버림 | `/clear` 후 재시작 |
| 대화가 20~30 턴 넘어감 | 응답 속도 느려짐 | `/clear` + 요약 복붙 |
| 새 작업 시작 | 다른 스킬/에이전트 작업으로 전환 | `/clear` |
| 에러 반복됨 | 같은 에러가 3번 이상 | `/clear` + 새 접근 |

### `/clear` 하는 올바른 순서

```
1. 먼저 요약 받기:
   "지금까지 만든 파일 목록과 주요 결정사항을 요약해줘."

2. 요약 내용 복사 (Cmd+C)

3. /clear 실행

4. 새 대화 시작 시 요약 붙여넣기:
   "이전 작업 요약:
   [붙여넣기]
   이어서 [다음 작업] 진행할게."
```

## 2. 모델 전환 (택시 등급 변경)

### 언제 어떤 모델 쓰나?

```bash
# 모델 변경 명령어
/model
```

| 작업 유형 | 추천 모델 | 이유 |
|----------|----------|------|
| 새 스킬 파일 처음 설계 | Opus (리무진) | 최고 품질 필요 |
| 에이전트 아키텍처 설계 | Opus | 복잡한 구조 이해 |
| 일반 콘텐츠 생성 | Sonnet (일반택시) | 가성비 최고 |
| 기존 스킬로 콘텐츠 반복 생성 | Haiku (경차) | 빠르고 저렴 |
| 간단한 수정·확인 | Haiku | 단순 작업 |

**절약 공식**: 스킬 설계는 Opus, 실제 사용은 Haiku → 비용 70% 절감

## 3. 세션 관리 (택시 나눠 타기)

### 작업 단위로 세션 분리

```
❌ 나쁜 예: 하나의 긴 대화에서 모든 것을 다 함
✅ 좋은 예: 작업별로 새 세션 시작
```

| 세션 | 작업 | 시작 전 |
|------|------|---------|
| 세션 1 | CLAUDE.md 작성 | claude 실행 → 작업 → /quit |
| 세션 2 | social-media.md 스킬 설계 | claude 실행 → 작업 → /quit |
| 세션 3 | 스킬 실행 + 콘텐츠 생성 | claude 실행 → 작업 → /quit |

## 4. 파일 참조 최적화

### 책상에 필요한 것만 올리기

```
❌ 나쁜 예:
"내 마케팅 폴더 전체를 분석해줘"
→ 수백 개 파일을 책상에 다 올림

✅ 좋은 예:
"@social-media.md 스킬에 링크드인 포스트 기능 추가해줘"
→ 딱 필요한 파일 하나만 참조
```

### @ 참조 vs 드래그앤드롭

| 방법 | 언제 | 토큰 비용 |
|------|------|----------|
| `@파일명` | 프로젝트 안 파일 | 적음 (경로만) |
| 드래그앤드롭 | 이미지/외부 파일 | 많음 (파일 내용 전체) |
| 전체 폴더 언급 | 절대 금지 | 매우 많음 |

## 5. 주의 신호 목록

### 이럴 때 즉시 `/clear` 하세요

- "이전에 말씀드린 대로..." 라는 응답이 나오는데 사실과 다름
- 응답 속도가 눈에 띄게 느려짐
- 같은 요청인데 품질이 갑자기 떨어짐
- "컨텍스트가 너무 길어서..." 라는 경고 메시지

### 이럴 때 새 프로젝트 폴더 고려

- 기존 CLAUDE.md와 다른 브랜드/프로젝트 작업할 때
- 실험적 스킬을 기존 시스템에 영향 없이 테스트할 때

## 6. Pro 플랜 일일 한도 관리

### 하루 사용량 배분 전략

```
오전 (신선한 한도):
→ 새 스킬 설계, 복잡한 에이전트 구조 작업

오후:
→ 기존 스킬 실행, 콘텐츠 생성 (Haiku로)

저녁:
→ 가벼운 수정, 검토
```

### 한도 초과 시 대처법

```
"Rate limit exceeded" 오류 발생 시:
1. Haiku 모델로 전환: /model → haiku 선택
2. 그래도 안 되면: 1~2시간 후 재시도
3. 매일 같이 초과된다면: Max 플랜 업그레이드 고려
```
