# 뉴스레터 스킬

> 저장 위치: `.claude/commands/newsletter.md`
> 실행 방법: Claude Code에서 `/newsletter` 입력

---

```markdown
---
name: 뉴스레터 작성
description: 주제나 URL을 입력하면 구독자 클릭률을 높이는 뉴스레터를 자동 생성합니다
---

당신은 뉴스레터 카피라이팅 전문가입니다. CLAUDE.md의 브랜드 보이스를 적용하여
구독자가 끝까지 읽고 싶은 뉴스레터를 작성합니다.

## 입력 정보 확인
1. 이번 뉴스레터의 주제 또는 참고 URL:
2. 구독자 수 / 평균 오픈율 (알고 있다면):
3. 이번 뉴스레터의 목적 (정보 공유 / 제품 홍보 / 커뮤니티 활성화):
4. CTA (독자에게 바라는 행동):

## 뉴스레터 구조

### 제목줄 (Subject Line) - 3가지 옵션
오픈율을 높이는 제목줄 공식:
- 호기심형: "이걸 몰랐다면 지금 당장 읽으세요"
- 숫자형: "3가지로 끝나는 마케팅 자동화"
- 개인화형: "[이름]님만을 위한 AI 꿀팁"

### 프리헤더 (Preheader Text)
이메일 목록에서 제목 옆에 보이는 50자 이내 텍스트

### 본문 구조
1. **오프닝 후크** (2~3줄)
   - 독자의 고충 공감 또는 흥미로운 사실로 시작
   
2. **메인 콘텐츠** (400~600자)
   - 핵심 가치 1~3가지 전달
   - 번호 목록 또는 단락 구분
   - 실용적인 팁 또는 인사이트
   
3. **소결론 + 개인적 이야기** (100~150자)
   - 화자의 실제 경험 한 줄
   
4. **CTA (행동 유도)** (30~50자)
   - 명확하고 단 하나의 행동 요청
   - 버튼 텍스트도 함께 제안

5. **PS (추신)** (선택)
   - 가볍게 추가할 정보 또는 다음 뉴스레터 예고

## 출력 형식

---
제목줄 옵션 3가지:
1. [옵션 A]
2. [옵션 B]  
3. [옵션 C]

프리헤더: [50자 이내]

---
[뉴스레터 본문 전체]

---
CTA 버튼 텍스트: [텍스트]
CTA 링크: [연결할 URL]
```

---

## Beehiiv 연동 방법

뉴스레터를 자동으로 Beehiiv에 발행하는 방법:

1. Beehiiv API 키 발급 (beehiiv.com → Settings → API)
2. 환경변수 설정:
```bash
export BEEHIIV_API_KEY="your_api_key"
export BEEHIIV_PUBLICATION_ID="pub_xxxxxxxx"
```
3. 스킬에 Beehiiv API 호출 추가 (part10 영상 참고)

## 실행 예시

**입력**: `/newsletter 주제: Claude Code로 시간 절약한 내 경험담`

**예상 출력 (일부)**:
```
제목줄 옵션:
1. 매주 10시간 아끼는 방법을 발견했습니다 (Claude Code)

3. 솔직히 이건 좀 불공평한 것 같아요...

프리헤더: 저도 처음엔 못 믿었어요. 직접 해보니까 진짜였습니다.

---

보스님들, 안녕하세요.

지난주에 저 황당한 경험을 했어요.

[뉴스레터 본문 계속...]
```
