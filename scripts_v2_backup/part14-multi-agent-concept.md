바이올린(소셜미디어), 첼로(리퍼포징), 트럼펫(SEO), 드럼(GA4), 피아노(브랜드 보이스).

근데 악기 하나만 있으면 독주밖에 못 하잖습니까. 독주도 멋지지만, 진짜 소름 돋는 건 오케스트라거든요.

스무 명의 연주자가 하나의 음악을 만들어내는 그 순간!

이제부터 이 악기들을 한 무대에 올려서 나만의 '콘텐츠 마케팅 오케스트라'를 완성할 겁니다.

하나씩 해봅시다!

## [CONCEPT] 독주 vs 오케스트라 – 왜 팀이 필요한가?

[📺 화면: 슬라이드]
[📊 PPT 지문: 독주 vs 오케스트라]

레스토랑으로도 비유해볼게요.

처음엔 사장님이 요리도 하고, 서빙도 하고, 계산도 하고, 청소도 합니다. 손님 3명까진 가능하죠.

근데 손님이 10명 넘으면? 요리 설거지하다가 서빙 놓치고, 계산 밀리고, 아무것도 제대로 못 합니다.

AI도 마찬가지거든요!

하나의 Claude Code에게 "리서치해줘, 콘텐츠 써줘, SEO 최적화해줘, 뉴스레터까지 써줘"를 한꺼번에 시키면, 지휘자 책상(컨텍스트)이 꽉 차고 품질이 떨어집니다.

독주, 그러니까 지금까지 했던 건 하나의 AI가 모든 것을 혼자 처리하는 방식이었습니다. 간단하고 빠르지만 복잡한 작업에서 품질이 떨어지죠.

[📊 PPT 지문: 독주 vs 오케스트라 비교
- 독주: 1 AI → 모든 작업 → 품질 저하
- 오케스트라: 전문 단원 역할 분담 → 퀄리티 UP]

오케스트라는 다릅니다. 여러 전문 단원이 역할을 나눠서 각 분야에서 전문화되니까 퀄리티가 확 올라갑니다. 그리고 지휘자가 전체를 조율하는 거죠.

## [CONCEPT] 우리 오케스트라 편성표

[📺 화면: 슬라이드]
[📊 PPT 지문: 우리가 만들 오케스트라]

자, 우리 오케스트라 편성표를 보시죠!

```
🎼 지휘자 (오케스트레이터)
│
├── 🔍 리서치 단원 (리서치 에이전트)
│   └── 주제 조사, 경쟁사 분석, 트렌드 파악
│
├── ✍️ 작곡가 단원 (콘텐츠 에이전트)
│   └── 블로그, 인스타, 뉴스레터 작성
│
├── 📊 음향 엔지니어 단원 (SEO 에이전트)
│   └── 키워드 최적화, 메타데이터 세팅
│
└── 📤 매니저 단원 (배포 에이전트)
    └── 발행 준비, 스케줄링
```

지휘자가 "오늘 주제는 'AI 마케팅'이야, 단원들 각자 역할 맡아서 해줘"라고 지시하면, 각 단원이 자기 전문 분야에서 최고 퀄리티로 작업합니다.

이게 바로 진짜 AI 마케팅 오케스트라입니다!

그런데 중요한 게 하나 있습니다. 에이전트를 만들기 전에 먼저 context 폴더에 브랜드 정보를 넣어야 합니다. brand-voice-guide.md, style-guide.md, product-info.md 같은 파일들이죠. 이게 있어야 에이전트가 우리 브랜드를 제대로 이해하거든요.

## [CONCEPT] 언제 독주하고, 언제 합주하나?

[📺 화면: 슬라이드]

"파자마보스님, 그러면 이제 항상 오케스트라로 해야 하나요?"

아닙니다! 상황에 따라 다르거든요.

오케스트라, 즉 합주가 필요한 경우는 주간이나 월간 콘텐츠를 한꺼번에 만들 때, 리서치에서 작성, 최적화, 배포까지 이어지는 작업을 할 때, 여러 파일이 동시에 생성되는 작업을 할 때입니다.

[📊 PPT 지문: 언제 합주? 언제 독주?
- 합주: 주간/월간 콘텐츠, 다단계 파이프라인, 여러 파일 동시 생성
- 독주: 인스타 1개, 기존 글 수정, 간단한 질문]

반대로 독주로 충분한 경우는 인스타 포스트 하나 빠르게 만들 때, 기존 글 수정할 때, 간단한 질문 답 받을 때죠.

규칙은 간단합니다. 이번 작업이 여러 단계를 거치고 결과물이 여러 개면 합주하는 겁니다!

## [CONCEPT] 에이전트가 작동하는 원리 – 심플하게!

[📺 화면: 슬라이드]
[📊 PPT 지문: 지휘자 → 단원 흐름]

기술적으로 어떻게 작동하냐고요? 걱정 마세요, 심플합니다.

1. 지휘자가 큰 작업을 받음 ("이번 주 콘텐츠 만들어줘")
2. 작업을 악장(파트)별로 나눔
3. 각 단원에게 자기 파트 배분
4. 단원들이 순서대로 또는 동시에 연주
5. 지휘자가 결과물 모아서 최종 확인

이걸 Claude Code 스킬 파일로 구현할 수 있습니다.

"아니 이걸 제가 만들 수 있다고요?"

당연하죠! 다음 파트에서 바로 만들어봅시다. 스킬 파일 만드는 거랑 비슷하거든요. 이미 5개나 만드신 보스님들한테 이건 식은 죽 먹기입니다!
