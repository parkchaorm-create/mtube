# Part 03: 터미널 공포증 완치 + Plan Mode – 삽질 방지 보험

## 메타 정보
- **예상 길이**: 22분 (약 5,500~6,600자)
- **화면 구성**: 웹캠 → PowerShell/터미널(5개 명령어 실습) → PPT(Plan Mode 비유) → Claude Code(리허설 vs 본공연 비교) → 웹캠
- **핵심 비유**: 터미널 = 무대 뒤 통로(백스테이지), 명령어 = 이동/준비 동작, Plan Mode = 공연 전 리허설, Edit Mode = 바로 본공연

## [HOOK] 오프닝 후크 (30~60초)

[📺 화면: 웹캠 단독]

보스님들, 솔직히 말할게요.

터미널이 처음엔 진짜 무서워 보이죠?

까만 화면에 흰 글씨. "이거 잘못 건드렸다가 컴퓨터 망가지는 거 아냐?" 이런 생각 드시죠?

저도 딱 그랬습니다. 처음 터미널 열었을 때 아무것도 안 하고 그냥 닫아버렸거든요.

근데 지금은요? 이게 없으면 일을 못 합니다.

터미널은 뭐냐면, 공연장 무대 뒤 통로입니다. 백스테이지죠. 관객은 몰라도 되지만, 연주자는 이 통로로 무대에 올라가야 하잖아요. 우리는 연주자니까 이 통로 사용법만 알면 됩니다.

오늘 딱 5개 동작만 배우면, 이 통로가 가장 편한 길이 되거든요. 그리고 삽질 방지 보험까지 한 번에 들고 가겠습니다.

## [CONCEPT] 터미널이란? (2분 이내 설명)

[📊 PPT 지문: 터미널 = 무대 뒤 통로(백스테이지) / 마우스 클릭 = 정문 통과 / 터미널 = 백스테이지 직행 통로]

터미널을 쉽게 설명하면 이렇습니다.

여러분이 평소에 마우스로 폴더 더블클릭하고, 파일 드래그하고, 버튼 클릭하잖아요. 터미널은 그걸 글로 하는 겁니다.

예를 들어 "바탕화면에 marketing 폴더 만들어줘"를 마우스로 하면 바탕화면 우클릭, 새 폴더, 이름 입력, 이렇게 세 단계잖아요. 터미널로는 `mkdir marketing` 이렇게 한 줄이면 끝이거든요.

"아니 그러면 왜 굳이 터미널을 써야 해요? 마우스가 더 편하잖아요."

좋은 질문이죠. Claude Code가 이 터미널에서 실행되기 때문입니다. 오케스트라 음향 장비가 무대 뒤 통로에 있으니까, 이 통로로 가야 장비를 켤 수 있는 거거든요.

## [DEMO] 딱 5개 동작 마스터

[📺 화면: PowerShell/터미널 창 전체화면]

[📊 PPT 지문: 터미널 5대 명령어 — pwd / ls(dir) / cd / mkdir / claude]

자, 이제 백스테이지 이동법 딱 5개만 익히면 됩니다. 직접 따라하면서 배워요.

윈도우 사용자분들은 파워셸을 열어주세요. 윈도우 키를 누르고 "PowerShell"을 검색해서 실행하시면 됩니다. 맥 사용자분들은 Spotlight에서 "터미널"을 검색해서 실행하시면 되고요.

---

### 동작 1: `pwd` – "지금 나 어디에 있지?"

[📺 화면: PowerShell — pwd 입력 → 결과 출력]

파워셸에 `pwd`를 입력하고 엔터를 눌러보세요.

```bash
pwd
```

이렇게 경로가 출력되죠? 예를 들어 "C:\Users\내이름" 이런 식으로요. 지금 내가 백스테이지 어느 복도에 서 있는지 알려주는 겁니다. 일을 시작하기 전에 늘 이걸로 위치 확인하는 습관을 만드세요.

---

### 동작 2: `dir` – "이 복도에 뭐가 있지?"

[📺 화면: PowerShell — dir 입력 → 파일/폴더 목록 출력]

이번엔 `dir`을 입력하고 엔터를 눌러보세요.

```bash
dir
```

지금 서 있는 복도에 어떤 방과 물건이 있는지 목록으로 보여줍니다. 맥에서는 `ls`를 쓰면 같은 결과가 나오거든요.

---

### 동작 3: `cd` – "저 방으로 이동!"

[📺 화면: PowerShell — cd Documents 입력 → 경로 변경 확인]

이번엔 폴더 안으로 들어가볼게요. `cd Documents`를 입력하고 엔터를 눌러주세요.

```bash
cd Documents
```

커서 앞에 표시되는 경로가 바뀐 거 보이시죠? 이제 Documents 폴더 안으로 들어온 겁니다.

이전 복도로 돌아가고 싶을 때는 `cd ..`을 입력하면 됩니다. 점 두 개거든요.

```bash
cd ..
```

한 발짝 뒤로 가는 거죠.

---

### 동작 4: `mkdir` – "새 방 만들기"

[📺 화면: PowerShell — mkdir my-ai-marketing 입력 → dir로 확인]

이번엔 새 폴더를 만들어볼게요. `mkdir my-ai-marketing`을 입력하고 엔터를 눌러주세요.

```bash
mkdir my-ai-marketing
```

그다음에 `dir`을 입력해서 확인해보시면, 방금 만든 my-ai-marketing 폴더가 목록에 생긴 게 보일 겁니다. 이게 앞으로 우리 오케스트라 연습실이 될 폴더거든요.

---

### 동작 5: `claude` – "음향 장비 켜기!"

[📺 화면: PowerShell — claude 입력 → Claude Code 실행 화면]

마지막이자 가장 중요한 동작입니다. 그냥 `claude`라고 입력하고 엔터를 누르면 됩니다.

```bash
claude
```

이걸 치면 음향 장비가 켜지고, 오케스트라가 연주할 준비가 되는 겁니다.

---

## [DEMO] 5개 동작 종합 실습 – 연습실 세팅

[📺 화면: PowerShell 전체화면]

방금 배운 5개를 합쳐서 실제 연습실을 세팅해볼게요. 저 따라오세요!

```bash
pwd
```

경로 확인됐죠? Documents 폴더로 이동합니다.

```bash
cd Documents
```

오케스트라 연습실 폴더를 만들게요.

```bash

```

만든 폴더 안으로 들어갑니다.

```bash
cd my-ai-marketing
```

현재 위치 다시 확인해볼게요. "Documents\my-ai-marketing"이 나오면 정상이거든요.

```bash
pwd
```

이제 음향 장비를 켜겠습니다.

```bash
claude
```

[📺 화면: Claude Code 실행 완료 화면]

어때요? 생각보다 쉽죠? 보스님들, 방금 오케스트라 연습실을 만들고 음향까지 켰습니다!

## [BONUS] 알면 편한 추가 팁 2가지

[📺 화면: PowerShell — 팁 시연]

보너스로 두 가지만 더 알려드릴게요.

첫 번째, Tab 키 자동완성입니다. 폴더 이름 치다가 Tab 키를 누르면 자동으로 완성되거든요. 예를 들어 "cd Doc"까지만 치고 Tab 키를 누르면 "cd Documents\"로 자동완성이 됩니다. 오타 줄이는 꿀팁이죠.

두 번째, 위 화살표 키입니다. 키보드 위쪽 화살표를 누르면 이전에 입력했던 명령어가 다시 불러와지거든요. 같은 명령어 반복할 때 타이핑 안 해도 됩니다.

---

## [CONCEPT] Claude Code 내부 명령어 – 음향 장비 버튼들

[📊 PPT 지문: Claude Code 내부 명령어 — /help / /clear / /model / /plan / /quit]

Claude Code 실행하고 나서 쓸 수 있는 명령어도 몇 가지 있습니다. 이건 파워셸 명령어가 아니고, Claude Code 안에서 쓰는 명령어거든요. 음향 장비의 버튼들이라고 생각하세요.

슬래시로 시작하는 게 포인트입니다. `/help`는 모든 버튼 설명서, `/clear`는 대화 초기화, `/model`은 AI 모델 변경, `/plan`은 리허설 모드, `/quit`은 장비 끄기입니다.

자, 이 중에서 `/plan`이 특히 중요하거든요. 지금 바로 알려드리겠습니다.

## [CONCEPT] Plan Mode – 삽질 방지 보험 1호

[📺 화면: 웹캠 단독]

보스님들, 삽질 방지 보험 얘기를 해야 합니다.

저 처음에 Claude Code 쓸 때 이런 경험 했거든요.

"소셜미디어 자동화 스킬 만들어줘"라고 했더니, Claude가 막 파일을 20개씩 쭉쭉 만들어버리는 겁니다. 근데 제가 원하던 게 아닌 거죠. 수정하려고 하니까 이미 다 얽혀있고, 처음부터 다시 해야 하는 상황이 됐습니다.

이게 뭔가 하면요, 오케스트라가 리허설 없이 바로 본공연을 시작한 겁니다! 지휘자가 "자, 시작!"했는데 바이올린은 모차르트 하고, 첼로는 베토벤 하고 있는 거죠.

리허설 한 번만 했으면 이런 일이 없었을 겁니다.

---

## [CONCEPT] Plan Mode vs Edit Mode

[📊 PPT 지문: Edit Mode(기본) = 바로 본공연 / Plan Mode = 리허설 먼저, OK하면 본공연 시작]

Claude Code에는 두 가지 모드가 있습니다.

기본 모드는 Edit Mode입니다. Claude가 바로 파일을 만들고 수정하죠. 빠르지만 원하지 않는 결과가 나올 수 있거든요.

Plan Mode는 리허설 먼저 하는 모드입니다. Claude가 먼저 "이렇게 연주할게요, 맞죠?"라고 계획을 보여줍니다. 악보 확인 후 승인하면 그때 본공연을 시작하는 거죠.

쉽게 말하면, Plan Mode는 리허설 먼저 보여주고 괜찮으면 본공연 시작하는 모드입니다.

"아니 그러면 매번 Plan Mode 써야 하나요?"

아닙니다! 간단한 수정은 바로 본공연, Edit Mode로 해도 됩니다. 새 악기 만들거나 복잡한 일 할 때만 리허설, Plan Mode를 하면 되거든요.

## [DEMO] Plan Mode 켜는 방법 – 따라해보세요

[📺 화면: PowerShell — Claude Code 실행 중]

Plan Mode 켜는 방법은 세 가지입니다.

첫 번째, 가장 쉬운 방법은 그냥 말로 하는 거죠. Claude Code 입력창에 이렇게 입력해보세요.

```
소셜미디어 자동화 스킬을 만들고 싶어.
근데 바로 만들지 말고, 계획을 먼저 보여줘. 내가 OK하면 그때 실행해.
```

"계획 먼저 보여줘"라는 한마디면 됩니다. 이게 제일 쉽거든요.

두 번째 방법은 슬래시 명령어입니다.

```
/plan
```

세 번째 방법은 안티그래비티 화면 아래쪽에 "Plan" 또는 "Edit"이라고 보이는 토글 버튼을 클릭하는 겁니다. 클릭 한 번으로 모드 전환이 되거든요.

## [DEMO] 리허설 있을 때 vs 없을 때 – 직접 비교

[📺 화면: PowerShell — Claude Code 실행 중]

직접 비교해볼게요.

먼저 리허설 없이 바로 본공연하는 경우입니다.

```
소셜미디어 콘텐츠 자동화 스킬 만들어줘
```

[📺 화면: Claude Code — 바로 파일 생성 시작]

보이시죠? Claude가 바로 파일들을 만들기 시작합니다. 어디로 가는지 모르겠고, 중간에 방향 바꾸기가 어렵거든요.

이번엔 리허설 먼저 하는 경우입니다. `/clear`로 초기화하고요.

```
/clear
```

이렇게 입력해보세요.

```
소셜미디어 콘텐츠 자동화 스킬을 만들 거야.
인스타그램과 유튜브 쇼츠 콘텐츠를 생성하는 스킬이야.
계획을 먼저 보여줘. 내가 OK하면 실행해.
```

[📺 화면: Claude Code — 계획 목록 응답]

보시죠? 이렇게 계획이 먼저 나옵니다. 어떤 파일을 어디에 만들 것인지, 어떤 기능을 넣을 것인지 목록으로 보여주거든요.

여기서 "링크드인도 추가해줘"라고 수정 요청도 가능합니다.

```
좋아, 근데 링크드인 포스트도 추가해줘.
```

Claude가 수정된 계획을 다시 보여주거든요. OK하면 그제서야 실행이 시작됩니다.

```
OK, 실행해줘.
```

리허설 한 번으로 삽질 80% 제거!

## [CONCEPT] 언제 리허설하고, 언제 바로 공연하나?

[📊 PPT 지문: Plan Mode 필수(새 악기, 여러 파일, 복잡한 설계) vs Edit Mode OK(단순 수정, 한두 줄 변경, 간단한 질문)]

규칙 딱 하나만 기억하세요.

"처음 만드는 거면 리허설!"

새로운 스킬 파일을 처음 만들 때, 여러 파일에 영향을 줄 때, 복잡한 걸 설계할 때는 반드시 Plan Mode를 켜세요. 단순 문장 수정이나 간단한 질문은 바로 Edit Mode로 해도 되거든요.

## [DEMO] 실수 복구 – 되돌리기 기능

[📺 화면: PowerShell — Claude Code 실행 중]

보너스입니다! 리허설 없이 공연했다가 망했을 때 대처법도 있거든요.

```
/rewind
```

이렇게 하면 최근 변경사항을 취소해줍니다. 공연 중에 틀린 음이 나왔을 때 "다시!"하고 처음으로 돌아가는 것과 같습니다. Esc 키를 두 번 눌러도 현재 작업을 중단할 수 있거든요.

겁먹지 마세요. 실수해도 되돌릴 수 있습니다.
