## [CONCEPT] prompt.md 기획서 – 공연 기획서부터 써라

[📺 화면: 슬라이드]
[📊 PPT 지문: prompt.md = 공연 기획서]

지휘자가 무대에 서기 전에 뭘 합니까? 악보를 봅니다. 근데 악보만 보는 게 아니라, 공연 기획서도 봅니다. "오늘 공연의 컨셉은 뭐고, 관객층은 누구고, 어떤 분위기로 갈 건지" 이런 전체 그림이 담긴 기획서죠.

우리도 마찬가지입니다. 오케스트레이터를 만들기 전에 prompt.md라는 기획서를 먼저 작성하는 겁니다.

[📊 PPT 지문: prompt.md 기획서 패턴
1. 목적: 이 오케스트라가 해결할 문제
2. 단원 목록: 어떤 에이전트를 어떤 순서로 호출할지
3. 입력: 사용자가 뭘 넣는지 (URL? 주제? 키워드?)
4. 출력: 최종 결과물 목록
5. 제약 조건: 브랜드 보이스, 글자 수 제한 등]

이 기획서를 먼저 작성하면 두 가지 이점이 있습니다.

첫 번째, Claude Code한테 "이 기획서대로 오케스트레이터 파일 만들어줘"라고 시키면 한 번에 완성도 높은 파일이 나옵니다. 일일이 수정할 필요가 없죠.

두 번째, 나중에 오케스트레이터를 수정하거나 다른 팀을 만들 때 이 기획서를 템플릿으로 재활용할 수 있습니다.

## [DEMO] prompt.md 기획서 작성 – 따라하세요!

[📺 화면: 안티그래비티 — 프로젝트 루트에 prompt.md 파일 생성]

프로젝트 루트에 `prompt.md` 파일을 만들겠습니다. 왼쪽 사이드바에서 프로젝트 이름 옆 새 파일 아이콘 클릭. `prompt.md` 입력. 그리고 아래 내용을 복붙하세요.

```markdown
# 마케팅 오케스트레이터 기획서

## 목적
URL 또는 주제 하나를 입력받아, 전체 마케팅 콘텐츠 패키지를 자동 생성한다.

## 단원 호출 순서
1. 플래너 → 전체 작업 순서 설계
2. 리서치 단원 → 주제 조사 + 트렌드 분석
3. 콘텐츠 단원 → 블로그 + SNS + 뉴스레터 작성
4. SEO 단원 → 키워드 최적화 + 메타 태그 생성
5. 배포 단원 → 발행 일정 + 체크리스트
6. 리뷰어 → 전체 결과물 품질 검수

## 입력
- 유튜브 URL 또는 콘텐츠 주제 (필수)
- 타겟 채널 (선택, 기본값: 전체)
- 발행 예정일 (선택)

## 출력 (outputs/ 폴더에 저장)
- research-notes.md
- blog-post.md
- instagram-posts.md
- newsletter.md
- x-thread.md
- distribution-schedule.md
- review-report.md

## 제약 조건
- context/brand-voice-guide.md 필수 참조
- 블로그 1,500자 이상
- 인스타 캡션 150자 이내
- 뉴스레터 600자 내외
```

이 기획서가 있으면 Claude Code한테 "이 prompt.md 기획서를 바탕으로 오케스트레이터 커맨드 파일을 만들어줘"라고 한 마디만 하면 됩니다. 직접 만드는 것보다 훨씬 정확한 결과물이 나오거든요.

## [DEMO] 지휘자 만들기 – 그대로 복붙하세요!

[📺 화면: 안티그래비티 — .claude/commands/marketing-team.md 생성]

기획서를 기반으로 만들어진 오케스트레이터 파일입니다. `.claude/commands/` 폴더에 `marketing-team.md`를 만드세요. 복붙하세요!

```markdown
---
name: AI 마케팅 팀 오케스트레이터
description: 주제 하나로 전체 AI 마케팅 오케스트라를 지휘하여 완성된 콘텐츠 패키지를 생성합니다
---

당신은 AI 마케팅 오케스트라의 지휘자(오케스트레이터)입니다.
아래 전문 단원들을 악장별로 조율하여 완전한 콘텐츠 패키지를 생성합니다.

## 단원 명단
- 📋 플래너: @.claude/agents/planner-agent.md
- 🔍 리서치 단원: @.claude/agents/research-agent.md
- ✍️ 콘텐츠 단원: @.claude/agents/content-agent.md
- 📊 SEO 단원: @.claude/agents/seo-agent.md
- 📤 배포 단원: @.claude/agents/distribution-agent.md
- 🔎 리뷰어: @.claude/agents/reviewer-agent.md

## 공연 전 확인 (먼저 물어보세요)
1. "오늘 다룰 콘텐츠 주제를 알려주세요"
2. "발행 타겟 채널을 알려주세요 (전체/특정 채널)"
3. "발행 예정일이 있나요?"

## 공연 순서 (STEP-BY-STEP)

### 서곡: 기획 (플래너)
"플래너로서 [주제]의 콘텐츠 전략을 수립해줘.
핵심 메시지, 타겟 채널, 작업 순서, 각 단원별 지시서를 작성해줘."

기획 완료 후 → 1악장으로

### 1악장: 리서치 (리서치 단원)
"리서치 단원으로서 플래너의 지시에 따라 [주제]를 조사해줘.
최신 트렌드 3가지, 독자 질문 5가지, 경쟁 콘텐츠 분석, 차별화 포인트 1가지 제공해줘."

리서치 완료 후 → 2악장으로

### 2악장: 콘텐츠 생성 (콘텐츠 단원)
리서치 결과를 전달하며:
"콘텐츠 단원으로서 위 리서치 결과를 바탕으로 콘텐츠를 작성해줘.
context/brand-voice-guide.md 브랜드 보이스 적용.
블로그 포스트 전체 + 인스타 2개 + 뉴스레터 생성해줘."

콘텐츠 완료 후 → 3악장으로

### 3악장: SEO 최적화 (SEO 단원)
"SEO 단원으로서 위 콘텐츠를 최적화해줘.
메타 태그 생성, 키워드 최적화, GEO 최적화 적용해줘."

SEO 완료 후 → 4악장으로

### 4악장: 배포 준비 (배포 단원)
"배포 단원으로서 위 콘텐츠를 발행 형식으로 정리해줘.
발행 일정과 체크리스트도 만들어줘."

### 앙코르: 품질 검수 (리뷰어)
"리뷰어로서 전체 결과물을 검수해줘.
브랜드 보이스, 맞춤법, SEO, 팩트 체크를 수행하고 합격/불합격 판정해줘."

불합격 시 → 해당 단원에게 수정 지시 → 재검수

### 커튼콜: 최종 보고
모든 악장 완료 후:
- 생성된 파일 목록
- 리뷰어 검수 결과
- 발행 준비 현황
- 다음 액션 체크리스트

## 결과물 저장
모든 결과물을 outputs/ 폴더에 저장:
outputs/
├── research-notes.md
├── blog-post.md
├── instagram-posts.md
├── newsletter.md
├── x-thread.md
├── distribution-schedule.md
└── review-report.md
```

[📺 화면: 파일 저장]

지휘자 탄생! 우리 오케스트라가 완성됐습니다!

## [DEMO] CLAUDE.md에 에이전트 라우팅 규칙 추가

[📺 화면: 안티그래비티 — CLAUDE.md 파일 열기]

지휘자를 만들었으니, 한 가지 더 해줄 게 있습니다. CLAUDE.md에 에이전트 라우팅 규칙을 추가하겠습니다.

마우스를 프로젝트 루트의 CLAUDE.md 파일로 가져가세요. 더블클릭해서 열어줍니다. 파일 맨 아래로 스크롤하시고, 아래 내용을 추가하세요.

```markdown
## 에이전트 라우팅 규칙
- 리서치/조사/분석 요청 → 리서치 에이전트 (.claude/agents/research-agent.md)
- 콘텐츠 작성/글쓰기 요청 → 콘텐츠 에이전트 (.claude/agents/content-agent.md)
- SEO/키워드/메타태그 요청 → SEO 에이전트 (.claude/agents/seo-agent.md)
- 발행/배포/스케줄 요청 → 배포 에이전트 (.claude/agents/distribution-agent.md)
- 품질 검수/리뷰 요청 → 리뷰어 에이전트 (.claude/agents/reviewer-agent.md)
- 전체 마케팅 패키지 요청 → /marketing-team 오케스트레이터 실행
```

이렇게 해두면 Claude가 작업 종류에 따라 적합한 에이전트를 자동으로 선택합니다. "SEO 최적화해줘"라고 하면 알아서 SEO 에이전트를 호출하는 거죠.

그리고 프로 팁 하나 알려드리겠습니다. Claude 공식 문서에서 서브에이전트 가이드와 스킬 가이드를 찾아서 프롬프트에 포함시키면 더 정확한 에이전트를 만들 수 있습니다. Claude Code 채팅창에 "Claude Code 공식 문서에서 subagent 가이드를 참고해서 에이전트 파일을 개선해줘"라고 입력하면 공식 문서를 참조해서 최적화해줍니다.
