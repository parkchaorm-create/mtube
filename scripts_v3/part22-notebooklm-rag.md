## [HOOK] 오프닝 후크

[📺 화면: 웹캠 단독]

보스님들, 솔직하게 말씀드리겠습니다.

AI한테 가장 큰 약점이 뭔지 아십니까? 기억력입니다. 매번 새 세션을 시작하면 이전에 뭘 했는지 싹 잊어버립니다. 건망증이 심한 거죠.

오케스트라로 비유하면, 바이올리니스트가 어제 연주했던 곡을 오늘 기억 못하는 겁니다. 매번 악보를 처음부터 다시 읽어야 하는 거죠. 이러니까 시간도 낭비되고, 토큰도 낭비됩니다.

근데 이걸 해결하는 방법이 있습니다. 구글의 NotebookLM을 연결하는 겁니다.

NotebookLM은 뭐냐면, AI를 위한 영구 도서관입니다. 도서관에 책을 넣어두면 언제든 꺼내볼 수 있잖습니까. NotebookLM이 바로 그 역할을 합니다.

공연이 끝날 때마다 연주 기록을 도서관에 보관하는 겁니다. 다음 공연 때 "저번에 어떻게 연주했더라?" 하면 도서관에서 꺼내오면 되는 거죠.

## [CONCEPT] NotebookLM = 영구 도서관

[📺 화면: 슬라이드]
[📊 PPT 지문: NotebookLM이 해결하는 3가지
1. 영구 기억 — 결정사항, 맥락이 세션 넘어 유지
2. 토큰 절약 — 매번 전체 맥락 로딩 불필요, 필요한 것만 가져옴
3. 새로운 능력 — 딥리서치, 인포그래픽, 팟캐스트 자동 생성]

NotebookLM이 Claude와 만나면 세 가지가 바뀝니다.

첫 번째, 영구 기억입니다. 지금까지는 세션이 끝나면 대화 내용이 사라졌습니다. NotebookLM에 저장하면? 다음 세션에서 "저번에 결정한 브랜드 전략 뭐였지?" 물어보면 바로 가져옵니다.

두 번째, 토큰 절약입니다. 여기가 핵심인데요. 매번 100페이지짜리 브랜드 가이드를 통째로 AI한테 읽히면 토큰이 확 나갑니다. NotebookLM 방식은 다릅니다. 필요한 정보만 쏙 뽑아오는 거죠. RAG라는 기술인데요, 도서관에서 책 전체를 읽는 게 아니라 필요한 페이지만 찾아서 펼치는 겁니다.

RAG, 쉽게 설명하겠습니다. 해리포터 시리즈 전권을 다 읽을 필요 없이, "볼드모트 관련 부분만 찾아줘" 하면 그 부분만 가져오는 기술입니다. 똑같은 결과를 훨씬 적은 비용으로 얻는 거죠.

세 번째, 새로운 능력입니다. NotebookLM은 인포그래픽도 만들어주고, 팟캐스트도 만들어주고, 딥리서치도 해줍니다. 그것도 무료로요.

## [DEMO] NotebookLM 연동 설정

[📺 화면: 안티그래비티 - notebooklm-skill.md 파일]

연동 방법을 알려드리겠습니다.

NotebookLM은 공식 API가 아직 없습니다. 비공식 Python 스크립트와 쿠키 인증을 사용하는 방식입니다. 어렵게 들리시죠? 걱정 마세요. Claude Code가 다 해줍니다.

먼저 스킬 파일을 만들겠습니다. `.claude/commands/notebooklm.md` 파일을 만드세요.

```markdown
# NotebookLM 연동 스킬

## 목적
NotebookLM과 Claude Code를 연결하여 영구 기억 시스템을 구축합니다.

## 설정 순서
1. notebooklm-api 파이썬 패키지 설치 (GitHub 레포에서)
2. NotebookLM 웹사이트 로그인 → 쿠키 토큰 획득
3. 토큰을 .env 파일에 저장
4. 연동 테스트: "내 노트북 목록 보여줘"

## 활용
- 프로젝트 기억 저장/검색
- 딥리서치 요청
- 인포그래픽 생성 요청
```

[📺 화면: 터미널에서 Claude Code 실행]

이 스킬 파일을 Claude Code에 넣고 "이 스킬 실행해줘"라고 하면, Claude Code가 알아서 패키지를 설치하고 인증 절차를 안내해줍니다.

브라우저 창이 열리면서 NotebookLM에 로그인하라고 합니다. 로그인하면 끝이거든요. 인증이 완료됐다는 메시지가 나옵니다.

테스트해볼게요.

```
내 NotebookLM에서 최근 노트북 3개 제목 알려줘.
```

[📺 화면: 터미널에서 결과 확인]

노트북 목록이 쭉 나옵니다. 연결 성공입니다!

## [CONCEPT] Wrap-up 스킬 = 자동 도서관 저장

[📺 화면: 슬라이드]
[📊 PPT 지문: Wrap-up 스킬 흐름
세션 작업 → /wrap-up 실행 → 핵심 내용 자동 정리 → NotebookLM에 저장
→ 다음 세션에서 "저번 작업 뭐였지?" → NotebookLM에서 검색 → 즉시 복원]

Wrap-up 스킬이라는 걸 만들겠습니다. 공연 끝나고 무대 뒤에서 "오늘 공연 어땠는지" 기록하는 일지 같은 겁니다.

대화가 길어지고 유용한 정보가 많이 쌓였을 때, `/wrap-up`을 실행하면 됩니다. 그러면 이 스킬이 세션 전체를 훑어서 핵심 내용만 정리하고, NotebookLM에 자동으로 저장합니다.

"보스님의 두뇌"라는 노트북이 NotebookLM에 만들어지고, 거기에 세션 기록이 계속 쌓이는 겁니다.

---

## [DEMO] Wrap-up 스킬 만들기

[📺 화면: 안티그래비티 - wrap-up.md 파일]

`.claude/commands/wrap-up.md` 파일을 만드세요.

```markdown
# 세션 갈무리 스킬

## 실행 조건
세션 종료 전, 또는 대화가 길어졌을 때

## 수행 작업
1. 현재 세션의 핵심 결정사항, 작업 결과, 배운 점 요약
2. NotebookLM의 "보스님 브레인" 노트북에 날짜별로 저장
3. 다음 세션 시작 시 참고할 핵심 포인트 3개 정리
4. TODO 항목이 있다면 별도로 정리

## 저장 형식
- 제목: [날짜] 세션 요약
- 본문: 결정사항, 작업 결과, 다음 할 일
```

[📺 화면: 터미널에서 /wrap-up 실행]

실제로 실행해보겠습니다. 지금까지 작업한 내용이 있으니까 `/wrap-up`을 실행합니다.

세션 전체를 분석하고... NotebookLM에 저장 중이라는 메시지가 나오고... 완료!

NotebookLM에 가서 확인해보면, "보스님 브레인" 노트북에 오늘 날짜로 세션 요약이 저장돼 있습니다.

## [CONCEPT] 지식 선순환 루프

[📺 화면: 슬라이드]
[📊 PPT 지문: 지식 선순환 루프
제작 → 기억 → 강화 → 더 나은 제작 → 더 풍부한 기억 → ...
"쓸수록 똑똑해지는 시스템"]

이게 왜 강력하냐면, 선순환이 생기기 때문입니다.

콘텐츠를 제작합니다. Wrap-up으로 기록을 저장합니다. 다음 세션에서 그 기록을 참고해서 더 나은 콘텐츠를 만듭니다. 그 결과를 또 저장합니다.

제작 → 기억 → 강화 → 더 나은 제작. 이 루프가 계속 돌아갑니다.

쓰면 쓸수록 시스템이 똑똑해지는 겁니다. 바이올리니스트가 매 공연마다 실력이 느는 것처럼요.

반대로 NotebookLM 없이 Claude만 쓰면? 매번 처음부터 시작합니다. 아무리 많이 써도 실력이 안 느는 거죠. 매 공연마다 악보를 새로 외우는 바이올리니스트인 겁니다.
