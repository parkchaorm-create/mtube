## [CONCEPT] 에이전트 vs 스킬 – 판단이냐, 반복이냐

[📺 화면: 슬라이드]

[📊 PPT 지문: 에이전트 vs 스킬 분리 원칙
| 구분 | 에이전트 | 스킬 |
|------|---------|------|
| 핵심 | 판단이 필요한 작업 | 반복 실행하는 작업 |
| 비유 | 지휘자의 즉석 판단 | 악보에 적힌 연주법 |
| 예시 | "이 데이터 분석해서 전략 짜줘" | /social-media로 콘텐츠 생성 |
| 특징 | 매번 다른 결과 | 항상 동일한 형식·품질 |]

악기를 만들기 전에 한 가지 중요한 구분을 짚고 갈게요.

에이전트와 스킬은 다릅니다. 기억하실 건 딱 한 줄입니다. "판단이 필요하면 에이전트, 반복 실행이면 스킬."

에이전트는 지휘자의 즉석 판단과 같습니다. "이 상황에서 뭘 해야 할지" 생각하고 결정하는 거죠. 매번 다른 답이 나올 수 있습니다.

스킬은 악보에 적힌 연주법입니다. 한 번 만들어두면 같은 형식, 같은 품질로 반복 실행됩니다. 오늘 만들 소셜미디어 콘텐츠 생성이 딱 이겁니다.

## [CONCEPT] 스킬이란? – 한 번 만들면 계속 쓰는 악기

[📺 화면: 슬라이드]

[📊 PPT 지문: 스킬 = 한 번 만들면 계속 쓰는 악기
| 구분 | 일반 ChatGPT | Claude Code 스킬 |
|------|-------------|-----------------|
| 방식 | 매번 긴 설명 입력 | 스킬 파일 한 번 만들면 끝 |
| 반복 작업 | 처음부터 다시 설명 | /social-media 명령어 하나로 실행 |
| 결과물 | 퀄리티가 매번 달라짐 | 항상 동일한 퀄리티·형식 |]

Claude Code 스킬의 개념을 먼저 이해해볼게요.

일반적으로 ChatGPT를 쓸 때는 매번 긴 설명을 입력해야 합니다. 같은 작업을 반복할 때마다 처음부터 설명해야 하고, 결과물 퀄리티도 매번 달라질 수 있죠.

근데 Claude Code 스킬은 다릅니다. 악기를 한 번 만들어두면, /social-media 명령어 하나로 실행되고, 항상 동일한 퀄리티와 형식으로 결과가 나옵니다.

스킬 파일은 프로젝트 폴더 안 `.claude/commands/` 폴더에 마크다운 파일로 저장합니다. 악기 보관함이라고 생각하시면 됩니다.

```
my-ai-marketing/          ← 우리 연습실
├── CLAUDE.md              ← 총보
└── .claude/
    └── commands/          ← 악기 보관함
        ├── social-media.md   ← 오늘 만들 첫 번째 악기
        ├── repurpose.md      ← 두 번째 악기 (나중에)
        └── seo-audit.md      ← 세 번째 악기 (나중에)
```

## [CONCEPT] "95% 확신 질문" 패턴

[📺 화면: 슬라이드 → 웹캠]

[📊 PPT 지문: 95% 확신 질문 패턴 — "95% 확신이 들 때까지 한 번에 하나씩 질문해" / 스킬에 이 한 줄 넣으면 → AI가 먼저 확인하고 실행]

여기서 스킬을 만들 때 꼭 알아야 할 패턴 하나를 알려드리겠습니다.

"95% 확신 질문"이라고 부르는 건데요. 스킬 파일 안에 이 한 줄을 넣는 겁니다.

"95% 확신이 들 때까지 한 번에 하나씩 질문해."

이게 왜 중요하냐면요. AI가 추측으로 결과를 내는 걸 막아줍니다. 확실하지 않은 부분이 있으면 먼저 물어보고, 정보가 충분해질 때까지 질문한 다음에 실행하는 거죠.

마치 연주자가 지휘자한테 "이 부분 템포가 빠른 건가요, 느린 건가요?" 확인하고 연주하는 것과 같습니다. 추측으로 연주하면 사고가 나잖아요.

이 패턴은 앞으로 만들 모든 스킬에 넣을 겁니다.

## [DEMO] 바이올린(소셜미디어 스킬) 제작 – 핵심 파트, 그대로 복붙하세요!

[📺 화면: 안티그래비티에서 좌측 파일 탐색기 열기 → .claude 폴더 → commands 폴더 클릭 → 우클릭하여 "새 파일" 선택 → 파일명에 "social-media.md" 입력하는 장면]

안티그래비티에서 `.claude/commands/social-media.md` 파일을 만들겠습니다.

이 전체를 그대로 복사해서 붙여넣으세요. 한 글자도 빠짐없이 복사하시는 게 중요합니다.

```markdown
---
name: 소셜미디어 콘텐츠 생성
description: 주제나 URL을 입력하면 인스타그램, 유튜브 쇼츠, 블로그용 콘텐츠를 동시에 생성합니다
---

당신은 CLAUDE.md에 정의된 브랜드의 전담 소셜미디어 콘텐츠 전문가입니다.
CLAUDE.md의 브랜드 보이스, 타겟 고객, 채널별 가이드라인을 반드시 따르세요.

## 먼저 확인할 것
95% 확신이 들 때까지 한 번에 하나씩 질문하세요.
콘텐츠 주제 또는 URL이 제공되었는지 확인하세요.
제공되지 않았다면 먼저 물어보세요: "오늘 다룰 주제나 참고할 URL이 있으신가요?"

## 생성할 콘텐츠

### 인스타그램 포스트
형식:
- 첫 줄: 시선을 끄는 후크 (30자 이내)
- 본문: 핵심 내용 + 가치 제안 (120~200자)
- 마무리: CTA (30자 이내)
- 해시태그: 한국어 8개 + 영어 5개 = 13개
- 이모지: 3~4개 자연스럽게

### 유튜브 쇼츠 스크립트
형식:
- 후크 (0~5초): 강렬한 첫 마디 (20자 이내)
- 핵심 내용 (5~50초): 3가지 포인트, 각 10초
- CTA (50~60초): 구독/링크 유도

### 블로그 제목 3개
- SEO 최적화된 제목
- 각각 다른 앵글 (호기심형, 숫자형, 방법론형)

## 출력 시 주의사항
- 채널명을 헤더(##)로 구분
- 각 콘텐츠 끝에 "📌 다음 액션:" 항목 추가
- 한국어로 작성 (브랜드 요구사항에 따라)
```

[📺 화면: Ctrl+S (맥은 Cmd+S)로 저장하는 장면]

## [DEMO] 첫 연주 – 악기 실행하기!

[📺 화면: 터미널에서 my-ai-marketing 폴더로 이동한 상태]

이제 이 악기를 처음으로 연주해봅시다. 따라하세요.

터미널에서 my-ai-marketing 폴더에 있는지 확인 후 Claude Code를 켜주세요.

claude

파일명이 곧 명령어 이름입니다. social-media.md를 만들었으니까 /social-media로 실행됩니다. 이게 전부입니다.

## [DEMO] 악기 튜닝 – 결과물 개선하기

[📺 화면: 결과물 검토 → 수정 요청]

첫 연주가 완벽했나요? 아마 아닐 겁니다. 괜찮습니다. 악기는 튜닝하면서 완성되는 거니까요.

결과물 보고 마음에 안 드는 부분이 있으면 바로 말하세요.

```
"인스타 캡션이 너무 형식적인 것 같아. 더 친근하고 구어체로 바꿔줘.
그리고 마지막 CTA에 '댓글로 알려주세요!' 같은 댓글 유도 문구 넣어줘."
```

[📺 화면: Claude Code 수정 결과]

좋아진 버전이 나왔죠?

핵심 꿀팁입니다. 이렇게 개선된 내용이 마음에 들면, 그 지시사항을 악기 설계도인 스킬 파일에 반영하세요. 그러면 다음부터는 처음부터 좋은 결과가 나옵니다.

스킬 파일의 인스타 섹션에 이 줄을 추가하면 됩니다.

```
- 톤: 격식체 금지. 친구에게 말하듯 편하게.
- CTA: 반드시 댓글 유도 문구 포함
```
