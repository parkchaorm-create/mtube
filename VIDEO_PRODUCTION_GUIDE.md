# 영상 제작 규칙

이 문서는 **풀코스 영상을 실제로 만드는 방법**에 대한 규칙입니다.
스크립트 내용과는 별개이며, 제작팀(우리)이 따르는 규칙입니다.

**디자인 레퍼런스**: [`contexts/design-references/dan-koe-style-guide.md`](contexts/design-references/dan-koe-style-guide.md)

---

## 디자인 시스템 (Dan Koe 스타일)

### 색상 팔레트 (4색만)
```
배경: #0A0A0A (딥 블랙)
텍스트: #FFFFFF (순백)
강조색: #E53E3E (레드) — 핵심 키워드, 하이라이트, 화살표 끝
보조: #6B7280 (그레이) — 부연 텍스트, 비활성 요소, 라인
```

### 폰트 체계 (3단)
```
제목: Pretendard ExtraBold (48~72px) / 올캡스 영문 Inter Black
본문 키워드: Pretendard Bold (24~36px)
코드/경로: JetBrains Mono (18~24px, 그레이 배경 박스)
```
캘리그래피 사용 안 함.

### 레이아웃 규칙
- 여백 60%+, 가운데 정렬
- 개념 하나 = 한 페이지
- 텍스트 자체가 비주얼 (단어 2~3개를 크게)
- 라인 드로잉: 1~2px 얇은 흰색 선, 최소 도형만

## 캐릭터

- **김보스 = 2D 심플 라인 캐릭터** (실사 아님, AI 아바타 아님)
- 단, **오프닝 첫 5초에만 힉스필드 실사 얼굴** → 2D 캐릭터로 전환

- 리모션으로 캐릭터 + 말풍선 + 인포그래픽 합성 애니메이션

## 인포그래픽 제작 파이프라인

### 생성 방식: Remotion-First
```
스크립트 [📊 PPT 지문:] 마커
    → /parse-script → partXX-slides.json
    → Remotion 7종 템플릿 컴포넌트 자동 렌더링
    → MP4 클립 (영상용) 또는 PNG (학습 키트용)
```

- **python-pptx**: 사용 안 함
- **Remotion**: 메인 (인포그래픽 영상 + 정적 이미지 모두)
- **Playwright**: 화면 캡처 전용
- **나노바나나**: 일회성 비유 일러스트만 (반복 요소는 SVG 직접)

### 7종 Remotion 템플릿
1. **TitleSlide** — 파트 제목 (Pretendard ExtraBold + 파트 번호)
2. **ConceptSlide** — 개념 설명 (아이콘 + 키워드 + 비유)
3. **ProgressMap** — 4ACT 프로그레스 맵 (현재 위치 레드 강조)
4. **ComparisonSlide** — 비교표 (예: 무료 vs 유료)
5. **StepSlide** — 단계별 설명 (1-2-3 스텝)
6. **RecapSlide** — 핵심 정리 (3줄 요약)
7. **ScreenCapture** — 화면 캡처 삽입 (이미지 + 캡션)

### 디자인 토큰 (theme.ts)
```typescript
export const theme = {
  colors: {
    bg: '#0A0A0A',
    text: '#FFFFFF',
    accent: '#E53E3E',
    muted: '#6B7280',
  },
  fonts: {
    title: 'Pretendard, sans-serif',
    body: 'Pretendard, sans-serif',
    code: 'JetBrains Mono, monospace',
  },
  lineWidth: 1.5,
}
```

## 애니메이션 레퍼토리

| 효과 | 구현 | 용도 |
|------|------|------|
| 라인 드로잉 (Draw-on) | SVG stroke-dashoffset | 개념도, 플로우차트 |
| 텍스트 페이드인 | opacity + translateY | 기본 등장 |
| 순차 등장 (Stagger) | 각 요소 delay | 리스트, 비교표 |
| 핵심 강조 | scale + 레드 flash | 핵심 키워드 |
| 타이핑 효과 | 글자별 순차 | 코드/명령어 |
| glow 효과 | box-shadow + blur | 오프닝/아웃로 전용 |

## 프로그레스 맵 규칙

| 위치 | 표시 시간 | 내용 |
|------|----------|------|
| ACT 시작 | 5초 | 전체 맵 + "ACT N 진입" |
| Part 시작 | 3초 | 해당 Part 위치만 |
| ACT 종료 | 5초 | 체크마크 + "ACT N 완료!" |
| ACT 3 종료 | 10초 | "핵심 완성!" 특별 강조 |

디자인: Dan Koe "THE FUTURE CAREER PATH" 스타일 — 중앙 타임라인에서 가지가 뻗어나가는 형태. 현재 위치 = 화이트+레드, 완료 = 그레이 체크, 미래 = 어두운 그레이.

## B-Roll 유형별 제작

### 유형 1: 인포그래픽 (CONCEPT 구간)
- **대상**: `[📊 PPT 지문:]` 마커
- **도구**: Remotion
- **스타일**: Dan Koe — 다크 배경 + 흰색 라인 + 레드 강조
- **전환**: 90초 이상 블록 단위, 설명→실습 전환 시 김보스 캐릭터가 브릿지

### 유형 2: 김보스 캐릭터 장면
- **대상**: `[📺 화면: 웹캠 단독]`
- **도구**: 나노바나나(이미지) → 리모션(모션) + 일레븐랩스(음성)
- **포즈**: 설명, 질문, 놀람, 축하 등 최소 6~8종

### 유형 3: 컴퓨터 화면 녹화 (DEMO 구간)
- **대상**: `[📺 화면: 안티그래비티...]`
- **도구**: OBS/QuickTime 수동 녹화
- **후처리**: B&W 오버레이 프레임 + 줌인 하이라이트

### 유형 4: 결과물 데모 (오프닝 전용)
- **도구**: 리모션 3D 트랜지션 + glow 효과
- **퀄리티**: 최고

## 파일 구조

```
production/
├── character/              ← 김보스 2D 캐릭터 시트
├── images/                 ← 인포그래픽 이미지 (파트별)
├── audio/                  ← 일레븐랩스 나레이션 (파트별)
├── video/
│   ├── clips/              ← 리모션 렌더링 클립
│   ├── screen-recordings/  ← 화면 녹화
│   └── partXX-final.mp4   ← 최종 합성
├── scenes/                 ← 파싱된 씬 JSON
└── bgm/                    ← 배경음악
```
